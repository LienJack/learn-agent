1. Diagram type

Layered architecture, because the point is that the event log remains the fact source while different execution bodies receive different context projections and then join back at the parent.

2. Visual elements

Bottom layer: Event Log as a long base. Middle layer: three projection cards, Main Context, Subtask Summary, Review Context. Top layer: Parent Agent, Sub-Agent, Reviewer in isolated workspaces. Right side: structured results converge into Parent Agent Join. Highlight Event Log and Join.

3. Positive prompt

Create an in-article technical architecture sketch about context isolation and result joining in delegation.

Style: off-white paper, black hand-drawn ink linework, slightly uneven strokes, sparse pale-yellow highlights, editorial technical illustration.

Composition: layered architecture. Bottom horizontal Event Log base, lightly highlighted, represented as a log scroll or timeline. Middle layer has three cards: Main Context with goal and next step; Subtask Summary with evidence, scope, assumptions; Review Context with diff, plan, risks. Top layer has three isolated workspaces labeled Parent Agent, Sub-Agent, Reviewer. On the right, a funnel and arrows merge Sub-Agent results and Reviewer notes into Parent Agent Join, highlighted in pale yellow. Keep labels short.
