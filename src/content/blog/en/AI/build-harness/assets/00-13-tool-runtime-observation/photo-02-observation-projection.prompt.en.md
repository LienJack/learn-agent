1. Diagram Type
Input-process-output diagram, because the focus is that raw result must not enter the model directly; it must pass through Normalizer and be projected into multiple views.

2. Visual Element Checklist
Left: Raw Result node with small labels for stdout, stderr, diff, and file bytes. Middle: Normalizer funnel highlighted in pale yellow. Right: four outputs: Model View, User View, Session Fact, Artifact Ref. Model View uses a brain-chip card, User View uses a terminal window, Session Fact uses a ledger, Artifact Ref uses a folder. Leave a bottom caption area: "Observation is factual projection, not raw output."

3. Positive Image Prompt
Create an in-article technical explanation diagram about how raw tool results are projected into observation.

Style: off-white paper background, black hand-drawn pen linework with slightly uneven stroke width, a small amount of pale yellow highlight, editorial technical illustration, hand-drawn technical blog flowchart, clear, restrained, with an engineering sketch feel.

Composition: input-process-output. Left side: a large Raw Result card containing very short labels: stdout, stderr, diff, file bytes. Middle: a funnel-shaped Normalizer highlighted in pale yellow, representing classification, cleanup, structuring, and truncation. Right side: four output cards connected by arrows from the funnel: 1. Model View: brain-chip and next-step arrow, representing facts consumable by the next model round. 2. User View: terminal window and eye icon, representing user-facing display. 3. Session Fact: ledger and timestamp, representing auditable event. 4. Artifact Ref: folder and chain icon, representing full output reference.

Use faint circuit traces and engineering sketch guides in the background. Keep all labels short. Leave a bottom caption area for "Observation is not stdout". Overall, it should feel like a hand-drawn mechanism diagram in a technical blog, not a product UI or dashboard.

4. Negative Prompt
No photorealism, no 3D, no complex UI screenshots, no large code blocks, no dense tiny text, no complex tables, no cyberpunk, no neon colors, no dark background, no colorful cartoon style, no complex shadows, no excessive decoration, no more than 6 main nodes, do not make background circuit lines more prominent than the subject, do not generate long text.
