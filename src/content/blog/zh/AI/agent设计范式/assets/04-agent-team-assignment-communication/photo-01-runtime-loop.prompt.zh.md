1. 图解类型
环形闭环流程图，因为这张图要解释 Agent Team 如何把用户目标一路推进到验证、合并、重分配或升级。

2. 画面元素清单
7 个主节点：目标、任务拆解、候选选择、上下文包、执行通信、报告验证、合并/重分配。图标分别使用小旗、分解成卡片的文档、放大镜与名单、文件夹包裹、对话气泡与齿轮、盾牌勾选、分叉出口。用淡黄色高亮“报告验证”和“合并/重分配”。背景使用很淡的电路线和工程草图辅助线。底部留一句短句位置：“多 Agent 的核心不是更多模型，而是可回放的协作闭环”。

3. 正向图片提示词
画一张文章内技术解释图，主题是 Agent Team 的最小运行闭环。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：顺时针环形流程图。画面包含 7 个关键节点，并用手绘箭头连接成闭环。① 目标：小旗图标，表示用户目标进入系统。② 任务拆解：一张大文档拆成多张小卡片，表示 TaskGraph / TaskSpec。③ 候选选择：放大镜扫过 agent 名单，表示 capability、tool、scope、load 过滤。④ 上下文包：文件夹和少量引用卡片，表示最小充分上下文。⑤ 执行通信：齿轮旁有两个对话气泡，表示 agent 执行与结构化消息。⑥ 报告验证：盾牌和勾选标记，淡黄色高亮，表示 AgentReport 进入 verifier。⑦ 合并/重分配：一个分叉出口，左边合并到主线，右边回到候选选择，淡黄色高亮。

背景有很淡的电路线、节点连线和工程草图辅助线，但不要抢主体。每个节点只放很短的中文标签。底部留出一行手写中文金句位置：“多 Agent 的核心不是更多模型，而是可回放的协作闭环”。整体感觉像技术博客中的手绘机制图，不是产品 UI，不是正式架构图。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 7 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
