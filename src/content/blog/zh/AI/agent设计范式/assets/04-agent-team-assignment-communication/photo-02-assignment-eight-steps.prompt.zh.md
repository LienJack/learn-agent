1. 图解类型
水平管线，因为这张图要解释一个任务从目标拆解到执行授权，再到监控验证的八步分配流程。

2. 画面元素清单
8 个主节点：拆解、分类、候选、模式、Owner、上下文、授权、监控验证。图标分别使用卡片栈、标签、名单、分叉开关、胸牌、压缩文件夹、钥匙、雷达盾牌。用淡黄色高亮“模式”和“监控验证”。背景使用很淡的电路线和工程草图辅助线。底部留一句短句位置：“分配不是指派名字，而是创建可验收的执行权”。

3. 正向图片提示词
画一张文章内技术解释图，主题是 Agent Team 的任务分配八步流程。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：从左到右的水平管线。画面包含 8 个关键节点，并用手绘箭头连接。① 拆解：一张目标卡片拆成多张 TaskSpec 卡片。② 分类：标签图标，表示 task_type、risk_level、write_scope。③ 候选：名单和放大镜，表示选择 eligible agents。④ 模式：分叉开关图标，淡黄色高亮，表示 lead_push、lease、recursive、redundant 的选择。⑤ Owner：胸牌图标，表示 primary、reviewer、verifier。⑥ 上下文：压缩文件夹和引用卡片，表示 ContextGateway 编译最小上下文。⑦ 授权：钥匙或租约卡片，表示 DelegationBrief / TaskLease。⑧ 监控验证：雷达加盾牌，淡黄色高亮，表示 heartbeat、report、verifier、reassign。

背景有很淡的电路线、节点连线和工程草图辅助线，但不要抢主体。每个节点只放很短的中文标签。底部留出一行手写中文金句位置：“分配不是指派名字，而是创建可验收的执行权”。整体感觉像技术博客中的手绘机制图，不是产品 UI，不是正式架构图。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 8 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
