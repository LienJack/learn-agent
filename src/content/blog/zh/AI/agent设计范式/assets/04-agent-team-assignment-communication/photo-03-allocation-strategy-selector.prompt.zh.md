1. 图解类型
决策路径，因为这张图要解释不同任务形态如何选择 Lead Push、能力匹配、租约、递归委派、冗余分配或人工分配。

2. 画面元素清单
7 个主节点：TaskSpec、任务形态判断、Lead Push、能力匹配、Claim+Lease、递归委派、冗余/人工。图标分别使用合同卡片、菱形分流器、指挥箭头、放大镜名单、租约钥匙、层级树、三份报告与人类审批印章。用淡黄色高亮“任务形态判断”和“冗余/人工”。背景使用很淡的电路线和工程草图辅助线。底部留一句短句位置：“策略选择取决于风险、领域、权限和可恢复性”。

3. 正向图片提示词
画一张文章内技术解释图，主题是 Agent Team 如何根据任务形态选择分配策略。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：中心决策路径。左侧是 TaskSpec 合同卡片，箭头进入中间的菱形判断节点“任务形态”，该节点用淡黄色高亮。右侧分出 5 条短路径：① Lead Push：Lead 小圆点直接把任务箭头推给一个 agent。② 能力匹配：放大镜从多张 agent 名单卡里选中一张。③ Claim+Lease：任务看板、认领手形和钥匙租约卡。④ 递归委派：RootLead 到 DomainLead 再到 Worker 的小树。⑤ 冗余/人工：三份并排报告进入 verifier 盾牌，旁边有人类审批印章，淡黄色高亮。

背景有很淡的电路线、节点连线和工程草图辅助线，但不要抢主体。每个节点只放很短的中文标签。底部留出一行手写中文金句位置：“策略选择取决于风险、领域、权限和可恢复性”。整体感觉像技术博客中的手绘机制图，不是产品 UI，不是正式架构图。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 7 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
