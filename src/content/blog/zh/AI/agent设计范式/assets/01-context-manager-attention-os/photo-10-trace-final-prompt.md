画一张文章内技术解释图，主题是 Trace 保存可审计链路，而不是隐藏思维链。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘因果链路图，清晰、克制、有工程草图感。

构图：横向 accountable trace 链。每个节点是一张小证据卡，用箭头串成“依据 -> 决策 -> 动作 -> 验证 -> 下一步”的公开链路，右上角放一个被锁住的“hidden CoT”小盒子，表示不保存。

节点：
1. Evidence：放大镜卡片，表示证据来源。
2. Decision：路标卡片，表示选择。
3. Action：扳手卡片，表示执行。
4. Validation：勾叉卡片，表示验证结果。
5. Failure Node：断点卡片，表示失败位置。
6. Next Step：小旗卡片，表示继续方向。

高亮：用淡黄色强调 Evidence 和 Validation；hidden CoT 只画成小锁盒，不作为主流程。
背景：很淡的因果线、节点连线和工程草图辅助线，不要抢主体。
文字：节点标签短；不要生成长推理文字。

负向提示词：不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 8 个主节点，不要画脑部扫描，不要表现私密思维内容。
