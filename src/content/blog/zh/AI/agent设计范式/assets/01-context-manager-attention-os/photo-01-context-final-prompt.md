画一张文章内技术解释图，主题是 Context 不是不断追加的 messages[]，而是 Agent 下一步行动所需的信息环境。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘机制图，清晰、克制、有工程草图感。

构图：分层架构图。左侧画一个很小的 messages[] 纸条堆，右侧画一个更完整的 Context Manager 工作台，工作台上有 6 个短标签模块，并用手绘箭头汇入“下一步判断”。

节点：
1. 用户目标：小旗图标，表示任务方向。
2. 当前状态：仪表盘图标，表示运行时现场。
3. 工具证据：扳手和回执图标，表示 tool result。
4. 文件环境：文件夹图标，表示工作区状态。
5. 记忆摘要：小脑图标，表示长期记忆和 summary。
6. 上下文编译：漏斗加文档图标，表示把材料组织成模型输入。

高亮：用淡黄色强调“上下文编译”和右侧完整工作台；messages[] 只画成较小、较单薄的输入之一。
背景：很淡的电路线、节点连线和工程草图辅助线，不要抢主体。
文字：节点标签短；底部留一行空白位置，方便后期加句子“Context 是信息环境，不是聊天数组”。如果中文文字不稳定，则只留干净标签块。

负向提示词：不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 8 个主节点。
