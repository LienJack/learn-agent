1. 图解类型
input-process-output，因为这一节的核心是把 Agent 的原始经历编译成三类可治理能力资产，并用评估门槛决定是否升级。

2. 画面元素清单
7 个主节点：经历、事实源、记忆、技能、优化、评估门、发布；图标分别用事件纸带、账本、档案卡、工具书、调参旋钮、检查闸门、火箭小旗；淡黄色高亮“评估门”和“技能”；背景是很淡的电路线和工程草图辅助线；底部留一句短 caption。

3. 正向图片提示词
画一张文章内技术解释图，主题是 Agent 如何把经历编译成能力。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：input-process-output。左侧画一卷从对话、工具调用、错误、用户反馈中流出的“经历”事件纸带，进入中间的“事实源”账本。账本右侧分成三路：上路是“记忆”档案卡，表示保存事实；中路是“技能”工具书，用淡黄色高亮，表示保存做法；下路是“优化”调参旋钮，表示行为改进。三路再汇入一个“评估门”检查闸门，用淡黄色高亮，只有通过的箭头进入右侧“发布”小旗；没通过的箭头回到草稿区。画面包含 7 个关键节点，并用手绘箭头连接。

节点：
① 经历：对话气泡、工具回执、错误标记组成的长纸带。
② 事实源：不可变账本，表示 Episode / Trajectory Log。
③ 记忆：档案卡，表示事实资产。
④ 技能：工具书和扳手，用淡黄色高亮，表示流程资产。
⑤ 优化：调参旋钮，表示 prompt patch、retrieval policy、tool heuristic。
⑥ 评估门：检查闸门和基线尺，用淡黄色高亮，表示跑赢 baseline。
⑦ 发布：小旗和版本标签，表示 gated release。

高亮：用淡黄色强调“技能”和“评估门”两个节点。

背景：很淡的电路线、节点连线和工程草图辅助线，不要抢主体。

文字：节点标签短；底部可留出一行手写金句位置：“记忆保存事实，技能保存做法”。如果中文文字不稳定，则只留空白区域给后期添加。

整体感觉像技术博客中的手绘机制图，不是产品 UI，不是正式架构图。元素清晰，层次明确，箭头关系明确。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 8 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
