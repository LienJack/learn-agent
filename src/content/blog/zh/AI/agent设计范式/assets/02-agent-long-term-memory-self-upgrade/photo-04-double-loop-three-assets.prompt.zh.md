1. 图解类型
layered architecture，因为这一节需要同时表达双循环、治理层和三层资产，不适合画成单条线性流程。

2. 画面元素清单
上层 Online Action Loop、中层 Offline Learning Loop、底层 Governance Layer；左侧三层资产堆叠：Raw Experience、Compiled Context、Executable Knowledge；淡黄色高亮“Executable Knowledge”和“Governance Layer”。

3. 正向图片提示词
画一张文章内技术解释图，主题是 Agent Experience OS 的双循环和三层资产。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘架构图，清晰、克制、有工程草图感。

构图：纵向分层架构图。画面中央从上到下是三个宽模块：Online Action Loop、Offline Learning Loop、Governance Layer。Online Action Loop 里画小箭头串起 Intent、Retrieve、Context、Act；Offline Learning Loop 里画小箭头串起 Trajectory、Reflection、Skill、Eval；Governance Layer 在最底下托住两条循环，包含 Scope、Provenance、Safety、Version、Rollback。左侧再画三层资产堆叠：Raw Experience、Compiled Context、Executable Knowledge，像三个叠放的抽屉。

节点：
① Online Loop：快速行动循环。
② Offline Loop：慢速学习循环。
③ Governance：治理底座，用淡黄色高亮。
④ Raw Experience：不可变事件账本。
⑤ Compiled Context：可检索的记忆和反思。
⑥ Executable Knowledge：skill、runbook、prompt patch，用淡黄色高亮。

高亮：用淡黄色强调“Executable Knowledge”和“Governance Layer”，表达真正升级来自可测试资产，且必须被治理。

背景：很淡的电路线、节点连线和工程草图辅助线，不要抢主体。

文字：节点标签短；底部可留出一行手写金句位置：“长期成长来自第三层”。如果中文文字不稳定，则只留空白区域给后期添加。

整体感觉像技术博客中的手绘机制图，不是产品 UI，不是正式架构图。元素清晰，层次明确，箭头关系明确。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 8 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
