1. 图解类型
horizontal pipeline，因为这一节的重点是长期记忆在线读取和离线写入分成两条不同路径。

2. 画面元素清单
上下两条泳道：读路径和写路径；8 个主节点：请求、读规划、检索重排、上下文组装、轨迹、候选提取、验证冲突、提交索引；淡黄色高亮“上下文组装”和“验证冲突”。

3. 正向图片提示词
画一张文章内技术解释图，主题是长期记忆的读写分离。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：双泳道水平流水线。上方泳道标为“读路径”，从左到右画快速、轻量的在线路径；下方泳道标为“写路径”，从左到右画较慢、严格的离线学习路径。两条泳道中间用虚线隔开，右侧共同连接到 Memory Store，但不要让两条线混成一条。

节点：
① 请求：对话气泡，进入读路径。
② 读规划：小清单，表示判断需要什么记忆。
③ 检索重排：放大镜、档案卡和排序滑杆。
④ 上下文组装：上下文包，用淡黄色高亮。
⑤ 轨迹：事件纸带，进入写路径。
⑥ 候选提取：候选记忆卡片和漏斗。
⑦ 验证冲突：盾牌、检查勾和碰撞标记，用淡黄色高亮。
⑧ 提交索引：账本写入和索引标签。

高亮：用淡黄色强调读路径的“Context Assembly”和写路径的“Verify / Conflict”，表达在线快、离线严。

背景：很淡的电路线、节点连线和工程草图辅助线，不要抢主体。

文字：节点标签短；底部可留出一行手写金句位置：“在线使用，离线学习”。如果中文文字不稳定，则只留空白区域给后期添加。

整体感觉像技术博客中的手绘机制图，不是产品 UI，不是正式架构图。元素清晰，层次明确，箭头关系明确。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 8 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
