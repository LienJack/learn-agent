1. 图解类型
layered architecture，因为这一节强调长期记忆必须 typed，不同类型回答不同问题，也落在不同工程存储中。

2. 画面元素清单
7 个记忆类型卡片：Episodic、Semantic、Preference、Procedural、Reflective、Graph、Environment；右侧是三类存储底座：log、JSON/vector、graph/skill store；淡黄色高亮“Procedural”和“Graph”；背景保持工程草图感。

3. 正向图片提示词
画一张文章内技术解释图，主题是 typed long-term memory 的七类记忆。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘架构图，清晰、克制、有工程草图感。

构图：分层架构图。左侧是一组从上到下排列的 7 张记忆卡片，每张只有短标签和简单图标；右侧是三层工程存储底座，用箭头把不同记忆类型连接到对应存储。画面像一个带标签的记忆档案柜，但不要画成复杂表格。

节点：
① Episodic：事件纸带，表示发生过什么。
② Semantic：事实卡片，表示稳定事实。
③ Preference：用户滑杆，表示偏好。
④ Procedural：工具书和扳手，用淡黄色高亮，表示怎么做事。
⑤ Reflective：便签和回看箭头，表示经验教训。
⑥ Graph：关系节点网络，用淡黄色高亮，表示实体和时间关系。
⑦ Environment：小地图和 API 插头，表示环境经验。

高亮：用淡黄色强调“Procedural”和“Graph”，因为它们分别连接 skill store 和 temporal graph，是文章后续自我升级的关键。

背景：很淡的电路线、节点连线和工程草图辅助线，不要抢主体。

文字：节点标签短；底部可留出一行手写金句位置：“记忆必须 typed，才可能治理”。如果中文文字不稳定，则只留空白区域给后期添加。

整体感觉像技术博客中的手绘机制图，不是产品 UI，不是正式架构图。元素清晰，层次明确，箭头关系明确。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 8 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
