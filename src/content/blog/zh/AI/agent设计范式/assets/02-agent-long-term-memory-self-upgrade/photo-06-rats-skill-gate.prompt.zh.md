1. 图解类型
decision path，因为 RATS 原则本质上是候选 skill 上线前必须通过的四道质量门。

2. 画面元素清单
候选 skill 卡片依次穿过 Reusable、Actionable、Testable、Scoped 四道门；每道门有一个简单图标；淡黄色高亮“Testable”和“Scoped”；失败分支进入 discard / revise。

3. 正向图片提示词
画一张文章内技术解释图，主题是 skill 候选必须通过 RATS 四道质量门。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘决策图，清晰、克制、有工程草图感。

构图：从左到右的决策路径。左侧是一张“Candidate Skill”卡片，像带齿轮的小文档。它沿着箭头依次穿过 4 道检查门：Reusable、Actionable、Testable、Scoped。每道门上有短标签和图标；通过后进入右侧“Eval Harness”，失败则从下方虚线箭头进入“Revise”草稿箱。

节点：
① Candidate：候选 skill 卡片。
② Reusable：循环箭头，表示可复用。
③ Actionable：清单和手指，表示可执行步骤。
④ Testable：测试台和勾选，用淡黄色高亮。
⑤ Scoped：边界框和定位针，用淡黄色高亮。
⑥ Eval Harness：带基线尺的测试盒，表示进入评估。
⑦ Revise：草稿箱，表示不通过就修改或丢弃。

高亮：用淡黄色强调“Testable”和“Scoped”，因为不可测试或无作用域的 skill 最容易污染系统。

背景：很淡的电路线、节点连线和工程草图辅助线，不要抢主体。

文字：节点标签短；底部可留出一行手写金句位置：“好经验必须可复用、可执行、可测试、有边界”。如果中文文字不稳定，则只留空白区域给后期添加。

整体感觉像技术博客中的手绘机制图，不是产品 UI，不是正式架构图。元素清晰，层次明确，箭头关系明确。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 8 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
