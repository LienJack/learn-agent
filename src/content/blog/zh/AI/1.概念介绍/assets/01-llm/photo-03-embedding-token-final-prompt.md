1. 图解类型
语义地图，因为Embedding 的关键是相似 token 在空间中靠近。

2. 画面元素清单
1. Token: 离散小卡片，表示原始编号
2. 向量坐标: 坐标轴和数字点，表示 embedding
3. 相似靠近: 猫狗小图标成簇，表示语义距离近
4. 相距较远: 汽车图标在远处，表示语义差异
5. 语义检索: 放大镜找附近点，表示 RAG 相似度检索
高亮：向量坐标、相似靠近
背景：米白纸张、很淡的电路线和工程草图辅助线。

3. 正向图片提示词
画一张文章内技术解释图，主题是 embedding 把 token 放入语义空间。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：二维语义地图。不同 token 小卡片散落在坐标平面上，相似概念聚成小簇。 画面包含 5 个关键节点，并用手绘箭头连接。

节点：
① Token：离散小卡片，表示原始编号。
② 向量坐标：坐标轴和数字点，表示 embedding。
③ 相似靠近：猫狗小图标成簇，表示语义距离近。
④ 相距较远：汽车图标在远处，表示语义差异。
⑤ 语义检索：放大镜找附近点，表示 RAG 相似度检索。

高亮：用淡黄色强调 向量坐标、相似靠近。

背景：很淡的电路线、节点连线和工程草图辅助线，不要抢主体。

文字：节点标签短；底部可留出一行手写金句位置：“语义相似，可以变成空间距离接近”。如果中文文字不稳定，则只留空白区域给后期添加。

整体感觉像技术博客中的手绘机制图，不是产品 UI，不是正式架构图。元素清晰，层次明确，箭头关系明确。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 8 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
