1. 图解类型
水平流水线，因为Tokenization 是从原始文本到模型颗粒的转换链路。

2. 画面元素清单
1. 原文: 纸条上的一句话，表示人类看到的完整文本
2. 切块: 菜刀和分割线，表示 tokenization
3. Token: 一组小方块，表示模型颗粒
4. 编号: 小标签和数字，表示离散 ID
5. 模型入口: 漏斗进入芯片，表示后续计算
高亮：切块、Token
背景：米白纸张、很淡的电路线和工程草图辅助线。

3. 正向图片提示词
画一张文章内技术解释图，主题是 文字被切成 token 后进入模型。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：从左到右流水线，像切菜台一样把完整句子切成小块，再送入模型入口。 画面包含 5 个关键节点，并用手绘箭头连接。

节点：
① 原文：纸条上的一句话，表示人类看到的完整文本。
② 切块：菜刀和分割线，表示 tokenization。
③ Token：一组小方块，表示模型颗粒。
④ 编号：小标签和数字，表示离散 ID。
⑤ 模型入口：漏斗进入芯片，表示后续计算。

高亮：用淡黄色强调 切块、Token。

背景：很淡的电路线、节点连线和工程草图辅助线，不要抢主体。

文字：节点标签短；底部可留出一行手写金句位置：“token 是语言进入模型前的最小处理单元”。如果中文文字不稳定，则只留空白区域给后期添加。

整体感觉像技术博客中的手绘机制图，不是产品 UI，不是正式架构图。元素清晰，层次明确，箭头关系明确。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 8 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
