1. 图解类型
层叠架构，因为Transformer 由重复 block 处理 token 表示。

2. 画面元素清单
1. Token 向量: 小方块排成序列，表示输入表示
2. Attention: 多条连线互相指向，表示上下文关系
3. FFN: 小齿轮模块，表示前馈变换
4. 多层堆叠: 重复积木层，表示深层网络
5. 输出表示: 顶部发光 token，表示更新后的语义表示
高亮：Attention、多层堆叠
背景：米白纸张、很淡的电路线和工程草图辅助线。

3. 正向图片提示词
画一张文章内技术解释图，主题是 Transformer block 如何承载 LLM 计算。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：纵向层叠结构。底部 token 向量进入多层 block，每层包含注意力和前馈网络两个模块。 画面包含 5 个关键节点，并用手绘箭头连接。

节点：
① Token 向量：小方块排成序列，表示输入表示。
② Attention：多条连线互相指向，表示上下文关系。
③ FFN：小齿轮模块，表示前馈变换。
④ 多层堆叠：重复积木层，表示深层网络。
⑤ 输出表示：顶部发光 token，表示更新后的语义表示。

高亮：用淡黄色强调 Attention、多层堆叠。

背景：很淡的电路线、节点连线和工程草图辅助线，不要抢主体。

文字：节点标签短；底部可留出一行手写金句位置：“Transformer 让关系计算更容易并行扩展”。如果中文文字不稳定，则只留空白区域给后期添加。

整体感觉像技术博客中的手绘机制图，不是产品 UI，不是正式架构图。元素清晰，层次明确，箭头关系明确。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 8 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
