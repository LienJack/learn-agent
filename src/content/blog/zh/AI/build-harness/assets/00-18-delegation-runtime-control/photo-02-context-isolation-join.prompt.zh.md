1. 图解类型

Layered architecture，因为重点是 event log 作为事实源，向不同执行体投影不同上下文，最后回到父 Agent join。

2. 画面元素清单

底层是 Event Log 长条底座；中间三张投影卡片：主上下文、子任务摘要、审查上下文；上层三个角色：父 Agent、Sub-Agent、Reviewer；右侧是结构化结果汇聚到父 Agent Join。高亮 Event Log 和 Join。图标包括日志卷轴、投影灯、隔离小隔间、审查放大镜、汇聚漏斗。

3. 正向图片提示词

画一张文章内技术解释图，主题是 delegation 中的上下文隔离和结果合并。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘架构图，清晰、克制、有工程草图感。

构图：分层架构图。底部是一条横向 Event Log 底座，像日志卷轴或时间线，用淡黄色轻微高亮，表示事实源。中间层有 3 张投影卡片：① 主上下文：目标和下一步；② 子任务摘要：证据、范围、假设；③ 审查上下文：diff、计划、风险。上层对应 3 个执行体：父 Agent、Sub-Agent、Reviewer，每个执行体像一个隔离小工作间。右侧用汇聚漏斗和箭头把 Sub-Agent 结果、Reviewer 意见合并到“父 Agent Join”，Join 节点用淡黄色高亮。

背景：很淡的电路线、节点连线和工程草图辅助线，不要抢主体。节点标签短，避免长句。底部可留一句手写金句位置：“trace 保存事实，context 只投影当前需要”。整体感觉像技术博客中的手绘机制图，不是产品 UI，不是正式架构图。

4. 负向提示词

不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 8 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
