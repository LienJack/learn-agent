1. 图解类型

Horizontal pipeline，因为重点是把 delegation 从“再调一次模型”改写成一条受控工具调用管线。

2. 画面元素清单

画面从左到右 7 个节点：父 Agent、任务包、Validate、Permission、受控子 Agent、Observation、父 Agent Join。父 Agent 用小指挥台图标，任务包用文件夹图标，Validate 用勾选清单，Permission 用盾牌，受控子 Agent 用小工作台，Observation 用状态卡片，Join 用汇聚箭头。淡黄色高亮 Permission 和 Join。底部留一句短句位置：“分出去的是工作，不是控制权”。

3. 正向图片提示词

画一张文章内技术解释图，主题是 Delegation Runtime 把 sub-agent 委派变成受控工具调用。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：横向 pipeline。画面包含 7 个关键节点，并用手绘箭头从左到右连接：① 父 Agent：小指挥台和目标旗，表示主线控制者；② 任务包：文件夹和清单，表示 objective、scope、contract；③ Validate：勾选清单，表示结构和范围校验；④ Permission：盾牌和闸门，用淡黄色高亮，表示工具与风险审批；⑤ 子 Agent：小工作台，表示隔离执行体；⑥ Observation：状态卡片，表示结构化结果；⑦ Join：多条箭头汇入父 Agent，用淡黄色高亮，表示父 Agent 审阅合并。

背景：很淡的电路线、节点连线和工程草图辅助线，不要抢主体。节点标签短，最好只使用“父 Agent”“任务包”“校验”“权限”“子 Agent”“观察”“Join”。底部留出一行手写金句位置：“分出去的是工作，不是控制权”。整体感觉像技术博客中的手绘机制图，不是产品 UI，不是正式架构图。
