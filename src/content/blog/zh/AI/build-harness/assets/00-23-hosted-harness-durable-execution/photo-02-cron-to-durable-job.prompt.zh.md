1. 图解类型

Circular loop，因为这张图要表达 Cron 不是一次性命令，而是围绕 job、lease、session、resume 和 notification 持续推进的可恢复生命周期。

2. 画面元素清单

画面采用顺时针循环，包含 7 个节点：Cron、Job、Queue、Worker、Session、Pause/Resume、Notify。中央放一个小型 Agent 任务卡片。Cron 用日历时钟图标；Job 用任务卡片；Queue 用队列线；Worker 用小服务器；Session 用事件日志本并淡黄色高亮；Pause/Resume 用暂停和播放按钮；Notify 用消息气泡并淡黄色高亮。箭头形成闭环，表示任务可能完成，也可能暂停后恢复。背景用很淡的电路线和虚线 checkpoint。

3. 正向图片提示词

画一张文章内技术解释图，主题是 Cron 如何从“定时执行命令”升级为“创建可恢复的 durable job lifecycle”。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：环形流程图。画面中央是一张小型 Agent 任务卡片，写短标签“Agent Job”。围绕中心按顺时针放 7 个流程节点，并用手绘箭头连接成闭环：① Cron：日历 + 小钟，表示定时触发。② Job：任务卡片，表示创建幂等任务。③ Queue：排队线和小圆点，表示持久化等待。④ Worker：小服务器，表示领取 lease。⑤ Session：事件账本，表示记录事实源，用淡黄色高亮。⑥ Pause/Resume：暂停键和播放键，表示等待批准或恢复。⑦ Notify：消息气泡，表示通知用户，也用淡黄色高亮。

箭头要表达：Cron 不直接跑命令，而是创建 job；worker 只是临时推进者；session 保存事实；任务可以暂停、恢复或完成。节点标签短，底部留出一行手写中文金句位置：“Cron 创建任务，不直接执行世界”。整体感觉像技术博客中的手绘机制图，不是产品 UI，不是正式架构图。

4. 负向提示词

不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 8 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
