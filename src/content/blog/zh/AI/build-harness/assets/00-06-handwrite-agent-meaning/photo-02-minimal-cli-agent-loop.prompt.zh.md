1. 图解类型
环形流程图，因为这张图要展示手写最小 CLI Agent 如何围绕模型意图、工具执行和观察回填形成闭环。

2. 画面元素清单
中央是“小型 CLI Agent”控制台；6 个环形节点：构建输入、模型判断、工具意图、权限策略、工具执行、观察回填；外侧有真实项目文件夹和测试结果。高亮“权限策略”和“观察回填”。底部留一句金句空位。

3. 正向图片提示词
画一张文章内技术解释图，主题是“手写最小 CLI Agent 的运行闭环”。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：环形流程图。画面中央是一个简洁的 CLI Agent 控制台，像终端窗口和小仪表盘组合。围绕中心画 6 个节点，顺时针连接：① 构建输入：文档卡片和漏斗，表示 context builder。② 模型判断：小芯片和思考气泡。③ 工具意图：带 JSON 小卡片的箭头。④ 权限策略：盾牌和分叉门，使用淡黄色高亮。⑤ 工具执行：扳手、终端和文件夹。⑥ 观察回填：账本卡片和回流箭头，使用淡黄色高亮。右侧外圈放“真实项目”文件夹和测试勾号，工具执行节点与它相连。

高亮：用淡黄色强调“权限策略”“观察回填”，以及 observation 回到构建输入的箭头。

背景：很淡的电路线、节点连线和工程草图辅助线，不要抢主体。

文字：节点标签短；底部可留出一行手写金句位置：“模型提意图，系统管执行”。如果中文文字不稳定，则只留空白区域给后期添加。

整体感觉像技术博客中的手绘机制图，不是产品 UI，不是正式架构图。元素清晰，层次明确，箭头关系明确。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 8 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
