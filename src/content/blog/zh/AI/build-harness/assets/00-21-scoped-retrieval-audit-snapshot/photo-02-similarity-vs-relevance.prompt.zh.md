1. 图解类型
Decision path，因为这张图要对比“语义相似”到“任务相关”的判断升级，重点是相似候选必须经过任务、来源、时间、权限和行动价值判断。

2. 画面元素清单
左侧是相似候选堆，中央是分叉判断路径，右侧分成“可用证据”和“相似但不相关”。6 个节点：Query、相似候选、任务边界、来源权威、时间有效、可用证据。淡黄色高亮任务边界和可用证据。图标使用放大镜、纸片堆、路标、印章、时钟、证据卡。

3. 正向图片提示词
画一张文章内技术解释图，主题是 semantic similarity 和 retrieval relevance 的区别。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：决策路径图。左侧画一个“Query”放大镜，箭头指向一堆“相似候选”纸片。中间画一条手绘分叉路，依次经过 4 个判断门：① 任务边界：路标和目标靶心，用淡黄色高亮，表示是否服务当前修测试任务。② 来源权威：印章图标，表示当前 workspace、项目规则、verified memory 的权威差异。③ 时间有效：时钟和版本标签，表示 asOf 时间点和过期材料。④ 行动价值：小扳手和检查清单，表示能否支持下一步行动。右侧上方是“可用证据”卡片，用淡黄色高亮；右侧下方是“相似但无关”纸片堆，使用虚线箭头表示被排除。

高亮：用淡黄色强调“任务边界”和“可用证据”。

背景：很淡的电路线、节点连线和工程草图辅助线，不要抢主体。

文字：节点标签短；底部可留出一行手写金句位置：“相似只是候选，相关需要边界”。如果中文文字不稳定，则只留空白区域给后期添加。

整体感觉像技术博客中的手绘机制图，不是产品 UI，不是正式架构图。元素清晰，层次明确，箭头关系明确。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 8 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
