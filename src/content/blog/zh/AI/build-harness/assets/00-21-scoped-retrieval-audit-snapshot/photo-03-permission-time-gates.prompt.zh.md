1. 图解类型
Layered architecture，因为这张图要展示检索结果进入模型前的多层门控：数据源访问、模型可见性、时间边界和投影快照。

2. 画面元素清单
4 层结构：Knowledge Sources、Source Access Gate、Model Visibility Gate、Time Snapshot Gate、Model Context。节点包含知识源库、访问盾牌、可见性眼睛、时钟快照、模型工作台。淡黄色高亮 Model Visibility Gate 和 Time Snapshot Gate。底部可放一条小的 audit log 轨道。

3. 正向图片提示词
画一张文章内技术解释图，主题是检索结果进入模型前必须经过权限和时间边界。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：纵向分层架构图，从下到上 5 层。底层是“Knowledge Sources”：画数据库、文件夹、session 日志、memory 卡片。第二层是“Source Access”：盾牌门禁，表示系统是否允许读取来源。第三层是“Model Visibility”：眼睛加遮挡条，用淡黄色高亮，表示读到的内容不一定能给模型看。第四层是“Time Snapshot”：时钟加相机快照，用淡黄色高亮，表示按当时版本固定证据。顶层是“Model Context”：小工作台和几张短证据卡，表示最终进入模型的可见投影。右侧画一条细线连接到底部“Audit Log”账本，表示每道门的决策都会记录。

高亮：用淡黄色强调“Model Visibility”和“Time Snapshot”两层。

背景：很淡的电路线、节点连线和工程草图辅助线，不要抢主体。

文字：节点标签短；底部可留出一行手写金句位置：“能读取，不等于能暴露”。如果中文文字不稳定，则只留空白区域给后期添加。

整体感觉像技术博客中的手绘机制图，不是产品 UI，不是正式架构图。元素清晰，层次明确，箭头关系明确。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 8 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
