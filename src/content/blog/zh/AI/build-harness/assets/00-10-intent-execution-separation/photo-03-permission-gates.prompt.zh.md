1. 图解类型
决策路径图，因为权限被拆成工具可见性和单次调用审批两道门。

2. 画面元素清单
左侧是 Runtime State，中间第一道门是 Tool Visibility，之后是 Model 和 Tool Intent，第二道门是 Approval Gate，右侧分三条路径：Allow 到 Execute、Ask 到 Human、Deny 到 Observation。两道门用淡黄色高亮。

3. 正向图片提示词
画一张文章内技术解释图，主题是“权限不是最后的弹窗，而是工具可见性和单次审批两道门”。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：从左到右的决策路径。左侧节点是 Runtime State，图标是齿轮和策略纸张。第一道门是 Tool Visibility，画成半开的门，使用淡黄色高亮，表示模型本轮能看到哪些工具。门后是 Model 节点和 Tool Intent 申请单。接着进入第二道门 Approval Gate，画成带盾牌的闸门，使用淡黄色高亮。闸门右侧分成三条路径：Allow 到 Execute 扳手，Ask 到 Human 小人和问号，Deny 到 Observation 日志纸张。

箭头关系要清楚，整体像权限流转草图。背景有很淡的电路线和工程辅助线。每个节点只放短标签。底部留一行空白金句位置。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 8 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
