1. 图解类型
分层架构图，因为重点是模型层和 Runtime 层的责任边界。

2. 画面元素清单
上层是 Model，只能写申请单；中层是 Harness Runtime，包含 validate、permission、scheduler；下层是真实世界，包括文件系统、Shell、Git、测试。边界线用粗黑线表示，Runtime 和边界线用淡黄色高亮。

3. 正向图片提示词
画一张文章内技术解释图，主题是“模型提交申请单，Runtime 才能接触外部世界”。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：分三层的 layered architecture。最上层是 Model，画成脑形芯片和一张写着 Intent 的小申请单。中间层是 Harness Runtime，画成控制台和三枚小模块：Validate、Permission、Scheduler，整层用淡黄色轻微高亮。中间层下方画一条粗黑边界线，标注“执行边界”。最下层是真实世界，包含文件夹、终端窗口、Git 分支、测试勾叉四个小图标，表示文件系统、Shell、Git、测试环境。

箭头从 Model 到 Runtime 只传递申请单；从 Runtime 到真实世界才画执行箭头。背景有很淡的电路线和工程草图辅助线。文字标签保持短，不要长句。整体要体现“模型不直接拥有工具，系统才拥有执行权”。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 8 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
