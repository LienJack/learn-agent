1. 图解类型
横向管线图，因为文章核心是模型意图如何一步步变成受控执行并回到观察。

2. 画面元素清单
六个主节点：用户目标、模型、Intent、Validate、Approve、Execute、Observe，其中 Intent 和 Approve 用淡黄色高亮。右侧观察结果用箭头回到模型，形成下一轮循环。图标分别使用对话气泡、脑形芯片、申请单、放大镜、闸门、扳手、日志纸张。底部留一句短 caption 空位。

3. 正向图片提示词
画一张文章内技术解释图，主题是“模型只提出 Intent，系统负责校验、审批、执行和观察回填”。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：横向管线图，从左到右排列 7 个节点，并用手绘箭头连接。① 用户目标：对话气泡和小旗，表示“修复测试失败”。② Model：脑形芯片，只负责提出下一步。③ Intent：申请单图标，使用淡黄色高亮，表示结构化意图。④ Validate：放大镜检查表，表示 schema 和 runtime 校验。⑤ Approve：闸门或盾牌，使用淡黄色高亮，表示 allow / ask / deny。⑥ Execute：扳手和终端窗口，表示系统执行。⑦ Observe：日志纸张和回环箭头，表示结果回填到下一轮模型。

右侧从 Observe 画一条弧形箭头回到 Model，表示 Agent Loop。背景有很淡的电路线和工程草图辅助线。每个节点只放短标签，底部留出一行空白金句位置。整体感觉像技术博客里的手绘机制图，不是正式企业架构图。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 8 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
