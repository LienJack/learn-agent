1. 图解类型
输入-处理-输出图，因为重点是同一条事件事实链如何投影给模型、审计和重放。

2. 画面元素清单
左侧输入是一串事件：Intent、Decision、Execution、Observation。中间是 Event Log / Projection Layer。右侧三个输出：Model Context、Audit Trail、Replay Runner。Event Log 和 Projection Layer 用淡黄色高亮。

3. 正向图片提示词
画一张文章内技术解释图，主题是“完整事件事实链同时服务观察、审计和重放”。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：input-process-output。左侧画四张竖排事件卡片：Intent、Decision、Execution、Observation，用细线连接成一条事实链。中间画一个较大的 Event Log 盒子和 Projection Layer 漏斗，使用淡黄色高亮，表示系统保留完整事实并做不同投影。右侧分成三个输出节点：Model Context，图标是小上下文窗口；Audit Trail，图标是放大镜和日志卷轴；Replay Runner，图标是回放按钮和时间线。

从 Projection Layer 分三条箭头到右侧三个输出，体现同一事实链的不同用途。背景有很淡的电路线和工程草图辅助线。标签保持短，整体清晰克制。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 8 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
