1. 图解类型
分层架构图，因为核心机制是 core 从内置具体能力，转向通过 Plugin Host 接收外部能力，并把扩展收束到统一纪律。

2. 画面元素清单
三层结构：底层 Core Kernel，中层 Plugin Host，上层外部能力。关键节点包括内置污染、Plugin Host、Contract、Registry、Lifecycle、Hook Gate、Harness 纪律。用淡黄色高亮 Plugin Host 和 Harness 纪律。图标使用核心芯片、拼图、登记册、时钟、闸门、盾牌。底部留一句短 caption 空位。

3. 正向图片提示词
画一张文章内技术解释图，主题是“Plugin Host 让外部能力进入同一套 Harness 纪律”。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘机制图，清晰、克制、有工程草图感。

构图：分层架构图。底部画一个稳固的 Core Kernel 芯片，旁边有少量文件和事件日志图标。中间画一个淡黄色高亮的 Plugin Host 闸门，像一个带检查口的桥。上层画四个外部能力拼图块：Provider、Tools、Policy、Trace。外部能力不能直接连到 Core，必须先进入 Plugin Host。Plugin Host 内部画四个小模块：Contract 纸张、Registry 登记册、Lifecycle 时钟、Hook Gate 闸门。右侧画一个盾牌标签“Harness 纪律”，用淡黄色高亮。

箭头从外部能力指向 Plugin Host，再从 Plugin Host 指向 Core Kernel。画一条被划掉的虚线箭头，从外部能力直接指向 Core，表示禁止旁路。背景有很淡的电路线和工程草图辅助线。节点标签保持短，底部留出一行空白金句位置。整体像技术博客中的手绘架构图，不是正式企业架构图。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 8 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
