1. 图解类型
横向演化流水线，因为这张图要解释从 Prompt 到 Harness 的责任扩张，而不是展示某个单点组件。

2. 画面元素清单
5 个主节点：Prompt、ChatBot、Agent、Harness、真实任务。每个节点用短中文标签和简单图标表示：文档、对话气泡、循环齿轮、控制台、项目文件夹。高亮 Agent 和 Harness。背景为很淡的工程连线。底部留一句空白金句位置。

3. 正向图片提示词
画一张文章内技术解释图，主题是“Agent 不是一句 Prompt，而是责任逐层增加的运行系统”。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：横向流水线，从左到右画 5 个关键节点，并用手绘箭头连接。① Prompt：一张写着短规则的纸，表示只约束回答。② ChatBot：两个对话气泡，表示管理多轮消息。③ Agent：循环箭头包住小齿轮，表示 Loop、Tools、State 组合推进任务，使用淡黄色高亮。④ Harness：带护栏的控制面板，表示权限、日志、验证、恢复，使用淡黄色高亮。⑤ 真实任务：项目文件夹和测试标记，表示外部环境里的可执行任务。

背景有很淡的电路线、节点连线和工程草图辅助线，但不要抢主体。每个节点只放很短的中文标签。底部留出一行手写中文金句位置，后期添加文字。整体感觉像技术博客中的手绘机制图，不是产品 UI，不是正式架构图。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 5 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
