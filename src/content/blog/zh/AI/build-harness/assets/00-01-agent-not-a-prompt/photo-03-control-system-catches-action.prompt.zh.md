1. 图解类型
决策路径图，因为这张图要表达模型提议不会直接落地，必须被控制系统拦截和校验。

2. 画面元素清单
左侧模型提议，中间 Harness 控制关卡，右侧真实执行。关卡包含权限、沙箱、日志、验证 4 个短节点。高亮 Harness 关卡和验证节点。箭头有允许和拒绝两条路径。

3. 正向图片提示词
画一张文章内技术解释图，主题是“模型提出行动，Harness 决定行动如何安全落地”。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：从左到右的决策路径。左侧画“模型提议”：小模型芯片吐出一张 action card。中间画一个较大的 Harness 控制关卡，像带护栏的检查站，使用淡黄色高亮。关卡里有 4 个小节点：① 权限：钥匙图标。② 沙箱：边界框图标。③ 日志：卷轴图标。④ 验证：测试勾选图标，验证节点也用淡黄色强调。右侧画“真实执行”：终端窗口和项目文件夹。另有一条细箭头向下通往“拒绝/询问”，用小路标表示高风险动作被拦下。

背景保持很淡的电路线和草图辅助线。中文标签尽量短。底部留出空白金句位置。整体不是安全海报，而是工程系统解释图。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 7 个主节点，不要生成长段中文文字。
