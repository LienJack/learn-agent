1. 图解类型
环形循环图，因为核心是模型每一轮根据新观察继续推进任务。

2. 画面元素清单
中央是 Agent 控制核心；外围 6 个节点：当前现场、模型判断、行动意图、工具执行、观察结果、状态更新。高亮工具执行和状态更新。用箭头形成闭环，右侧画一个外部项目文件夹作为真实世界。

3. 正向图片提示词
画一张文章内技术解释图，主题是“Agent loop 如何把一次回答变成持续推进”。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：环形流程图。画面中央是一个简洁的 Agent 控制核心图标，像小控制盘或圆形齿轮。外围按顺时针画 6 个节点：① 现场：文件夹和小放大镜，表示当前任务上下文。② 判断：脑形线稿或模型芯片，表示模型决定下一步。③ 意图：小纸条和箭头，表示 tool call 或 action proposal。④ 执行：扳手和终端窗口，表示工具真正执行，使用淡黄色高亮。⑤ 观察：日志卷轴，表示工具输出。⑥ 更新：状态卡片叠放，表示把结果写回 state，使用淡黄色高亮。右侧加一个淡淡的项目文件夹，表示真实世界在循环外被访问。

背景有很淡的电路线和工程草图辅助线。节点标签短，避免长句。底部留出一行空白手写金句位置。整体像技术博客里的手绘机制图。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 6 个主节点，不要生成长段中文文字。
