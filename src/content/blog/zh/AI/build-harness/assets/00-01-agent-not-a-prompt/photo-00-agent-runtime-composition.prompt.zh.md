1. 图解类型
输入-处理-输出图，因为这张图要解释 Agent 不是单句 Prompt，而是五个运行部件组合成一个受控过程。

2. 画面元素清单
左侧 5 个输入模块：模型、循环、工具、状态、控制。中央一个淡黄色高亮的 Agent 运行工作台。右侧一个任务文件夹和检查标记，表示真实任务推进。高亮中央 Agent 和控制模块。背景使用很淡的工程连线。

3. 正向图片提示词
画一张文章内技术解释图，主题是“Agent 是模型、循环、工具、状态、控制系统组合起来的运行过程”。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘机制图，清晰、克制、有工程草图感。

构图：input-process-output。画面左侧分成 5 个独立输入模块，像五张手绘卡片或拼图块，汇入画面中央一个淡黄色高亮的“Agent 运行过程”工作台，右侧输出为“真实任务推进”的项目文件夹和检查标记。用手绘箭头表示五个部件不是一句 prompt，而是一起驱动运行过程。

节点：
① 模型：小脑/芯片图标，表示判断下一步。
② 循环：环形箭头图标，表示多轮推进。
③ 工具：扳手图标，表示接触外部世界。
④ 状态：层叠便签/日志图标，表示过程不断线。
⑤ 控制：护栏/盾牌图标，表示权限、预算、验证。
⑥ Agent 运行：中央工作台，淡黄色高亮，表示组合后的受控过程。
⑦ 任务推进：右侧项目文件夹 + 测试通过勾，表示真实任务被推进。

高亮：用淡黄色强调中央“Agent 运行”和“控制”模块。

背景：很淡的电路线、节点连线和工程草图辅助线，不要抢主体。

文字：只使用非常短的中文节点标签：模型、循环、工具、状态、控制、Agent、任务。不要生成长段中文文字。整体感觉像技术博客中的手绘机制图，不是产品 UI，不是正式架构图。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 7 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
