1. 图解类型
环形循环图，因为重点是 governance store 不是终点，而是需要持续使用信号、体检、过期、撤销、合并和复核的治理循环。

2. 画面元素清单
画面中央是 Governance Store 数据库，外围顺时针 6 个节点：检索使用、使用信号、健康检查、过期/降权、冲突复核、合并版本。使用数据库、放大镜、心电图、时钟、闸门、合并箭头等图标。淡黄色高亮健康检查和冲突复核。底部留一句金句位置。

3. 正向图片提示词
画一张文章内技术解释图，主题是 Agent 长期记忆治理存储的健康检查循环。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘循环图，清晰、克制、有工程草图感。

构图：环形循环图。画面中央是一个标注为 Governance Store 的数据库和小账本组合图标。外围按顺时针放 6 个节点，并用手绘箭头形成闭环：① 检索使用：放大镜和 query 气泡，表示 memory 被召回。② 使用信号：小统计点和眼睛图标，表示命中、忽略、帮助或误导。③ 健康检查：心电图和检查表，使用淡黄色高亮。④ 过期/降权：时钟和向下箭头，表示 expired / deprecated。⑤ 冲突复核：两张冲突卡片和闸门，使用淡黄色高亮。⑥ 合并版本：分支合并箭头和版本标签，表示 merge / supersede。

高亮：用淡黄色强调健康检查和冲突复核。

背景：很淡的电路线、节点连线和工程草图辅助线，不要抢主体。

文字：节点标签短；底部可留出一行手写金句位置：“长期记忆要维护，不是只新增”。如果中文文字不稳定，则只留空白区域给后期添加。

整体感觉像技术博客中的手绘机制图，不是产品 UI，不是正式架构图。元素清晰，层次明确，循环关系明确。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 8 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
