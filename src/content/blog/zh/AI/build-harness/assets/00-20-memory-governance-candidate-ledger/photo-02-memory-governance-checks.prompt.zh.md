1. 图解类型
决策路径图，因为重点是候选记忆必须经过来源、范围、TTL、冲突和 review gate 的分支判断。

2. 画面元素清单
画面中心是一条自上而下的判断路径：候选记忆、来源检查、范围检查、TTL 检查、冲突检查、Review Gate、写入或拒绝。旁边有三条分支：降置信度、等待更多证据、拒绝并留痕。使用盾牌、时钟、地图针、冲突闪电、闸门、数据库等图标。淡黄色高亮 Review Gate 和冲突检查。

3. 正向图片提示词
画一张文章内技术解释图，主题是 Memory Governance 对候选记忆的治理检查路径。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘决策图，清晰、克制、有工程草图感。

构图：决策路径图。画面上方是“候选记忆”卡片，向下进入一串判断节点，并用手绘箭头连接。节点包括：① 来源：小印章和证据纸片，表示来源可靠性。② 范围：地图针和文件夹，表示 user / repo / task scope。③ TTL：小钟表，表示过期和复核时间。④ 冲突：两张相撞卡片和小闪电，使用淡黄色高亮，表示与旧记忆冲突。⑤ Review Gate：闸门和人类审批小图标，使用淡黄色高亮。⑥ Store / Reject：一边是数据库，一边是归档盒，表示批准写入或拒绝留痕。

高亮：用淡黄色强调冲突检查和 Review Gate。

背景：很淡的电路线、节点连线和工程草图辅助线，不要抢主体。

文字：节点标签短；底部可留出一行手写金句位置：“相似不等于可信，可信不等于可用”。如果中文文字不稳定，则只留空白区域给后期添加。

整体感觉像技术博客中的手绘机制图，不是产品 UI，不是正式架构图。元素清晰，层次明确，分支关系明确。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 8 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
