1. 图解类型
分层架构图，因为这张图要把一轮模型输入呈现成整理好的工作台，而不是历史仓库。

2. 画面元素清单
中央是一张 Model Input 工作台，分 5 层：Trusted Rules、Current State、Latest Observation、Evidence Pack、Visible Tools。左侧有 Session/State 来源，右侧有 Model 判断。高亮 Trusted Rules 和 Latest Observation。

3. 正向图片提示词
画一张文章内技术解释图，主题是“修测试任务中模型这一轮真正看到的是一张整理好的工作台”。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：分层工作台图。中央画一张大工作台或叠层面板，标题短标签“Model Input”。面板分 5 层：① Trusted Rules：规则文档和盾牌，使用淡黄色高亮。② Current State：任务现场卡片。③ Latest Observation：测试日志节选和警示标记，使用淡黄色高亮。④ Evidence Pack：文件片段和引用链。⑤ Visible Tools：扳手集合。左侧画 Session Log 和 State 两个来源卡片，箭头流入工作台。右侧画 Model 芯片和“下一步”箭头。

背景有淡电路线和工程草图辅助线。每层只放短标签，不写长中文。底部留一行空白金句位置。整体像技术博客的手绘解释图。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 8 个主节点，不要生成长段中文文字。
