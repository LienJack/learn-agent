1. 图解类型
决策路径图，因为这张图要展示候选内容进入上下文前的多道检查门。

2. 画面元素清单
左侧候选内容卡片，依次经过 5 个 gate：相关性、当前事实、权限、信任、预算。每个 gate 有允许/排除/摘要分支。高亮信任和预算两个 gate。右侧输出 Model Input。

3. 正向图片提示词
画一张文章内技术解释图，主题是“不是相关就能进上下文，候选内容必须经过 Context Policy 的多道门”。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：从左到右的决策路径。左侧画一叠候选内容卡片，标签“候选内容”。中间连续画 5 个小闸门：① 相关性：放大镜。② 当前事实：时钟和刷新箭头。③ 权限：钥匙。④ 信任：盾牌，使用淡黄色高亮。⑤ 预算：token 计量尺，使用淡黄色高亮。每个 gate 下方有细小分支通向“排除 / 摘要 / 引用”，不要写长句。右侧画整理好的 Model Input 卡片。

背景有淡电路线和草图辅助线。中文标签保持短。底部留空白金句位置。整体要表现工程筛选，不是营销漏斗。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 7 个主节点，不要生成长段中文文字。
