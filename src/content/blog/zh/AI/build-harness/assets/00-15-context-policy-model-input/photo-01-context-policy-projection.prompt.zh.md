1. 图解类型
输入-处理-输出图，因为这张图要表达多种上下文来源经过 Context Policy 后才成为本轮 Model Input。

2. 画面元素清单
左侧 5 个输入节点：Session Log、State、Rules、Memory、Retrieval。中间是 Context Policy 工作台，包含选择、压缩、隔离、预算。右侧是 Model Input 和 Decision Ledger。高亮 Context Policy 与 Model Input。

3. 正向图片提示词
画一张文章内技术解释图，主题是“Context Policy 把事实、状态、记忆和检索结果投影成本轮模型输入”。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：输入-处理-输出布局。左侧画 5 张输入卡片：① Session Log：时间线事件卡。② State：当前现场卡片。③ Rules：项目规则文档。④ Memory：小书架。⑤ Retrieval：放大镜和证据片段。中间画一个较大的 Context Policy 工作台，使用淡黄色高亮，工作台上有四个短标签：选择、压缩、隔离、预算。右侧输出两个节点：Model Input，像整理好的模型窗口，使用淡黄色高亮；Decision Ledger，像小账本，表示投影记录。箭头从所有输入流向 Context Policy，再分流到 Model Input 和 Decision Ledger。

背景有很淡的电路线和工程草图辅助线。节点标签短，避免长段中文。底部留出一行空白金句位置。整体像工程机制图，不是 UI 截图。
