1. 图解类型
状态序列图，因为这张图要展示 Agent 责任从会回答逐步演化到可托管运行。

2. 画面元素清单
5 个阶段：Chat Agent、Tool Agent、Runtime Agent、Managed Agent、Harness。每个阶段有新增责任小图标。高亮 Runtime Agent 和 Managed Agent，因为它们是从 demo 走向真实系统的转折点。

3. 正向图片提示词
画一张文章内技术解释图，主题是“Agent 从 Chat Agent 自然演化到 Harness”。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：从左到右的状态序列。① Chat Agent：对话气泡和 messages 卡片。② Tool Agent：扳手和 tool schema 卡片。③ Runtime Agent：时钟、预算条、回放箭头，使用淡黄色高亮。④ Managed Agent：沙箱框、权限钥匙、trace 线，使用淡黄色高亮。⑤ Harness：外部控制台和护栏，表示模型外部控制系统。每个阶段下方只放 1-2 个短标签。箭头表示责任被现实压力一步步推出。

背景有淡电路线和工程草图辅助线。底部留出空白金句位置。整体不是产品路线图，而是工程演化机制图。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 6 个主节点，不要生成长段中文文字。
