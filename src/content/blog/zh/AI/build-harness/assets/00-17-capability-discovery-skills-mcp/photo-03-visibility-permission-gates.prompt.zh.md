1. 图解类型
decision path，因为这张图要解释候选能力先过可见性门，再过权限门，两道门控制的是不同空间。

2. 画面元素清单
7 个主节点：候选能力、可见性门、不可见分支、模型计划、权限门、拒绝 observation、执行与审计。用淡黄色高亮“可见性门”和“权限门”。图标使用目录、门、眼睛、脑袋/对话气泡、盾牌、回执、扳手与日志。

3. 正向图片提示词
画一张文章内技术解释图，主题是动态工具暴露里的两道门：可见性门和权限门。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘决策路径图，清晰、克制、有工程草图感。

构图：从左到右的分叉决策路径。左侧是“候选能力”目录图标，箭头进入第一道门“可见性门”，这个门用淡黄色高亮，门上画眼睛图标，表示任务、路径、预算、模式过滤。向下分支到“不可见”，用灰色小盒子表示不进入本轮 prompt。向右分支到“模型计划”，用简笔脑袋和对话气泡表示模型可以提出 intent。再进入第二道门“权限门”，用淡黄色高亮，门上画盾牌图标，表示参数、风险和用户审批。权限门向下分支到“拒绝 observation”，用回执图标；向右分支到“执行与审计”，用扳手和日志图标。

背景有很淡的电路线和工程草图辅助线，但不要抢主体。每个节点只放短中文标签。底部留出一行手写中文金句位置：“可见性控制计划空间，权限控制执行空间”。整体感觉像技术博客中的手绘机制图，不是产品 UI，不是正式架构图。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 7 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
