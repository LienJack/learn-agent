1. 图解类型
layered architecture，因为这张图要解释 Skills、MCP、ToolSearch、Deferred Loading 与统一 tool pipeline 的层级关系。

2. 画面元素清单
5 层结构：能力来源层、能力目录层、发现与搜索层、模型投影层、执行管线层。节点包含 Skills、MCP、ToolSearch、Deferred Loading、Visible Tools、Tool Pipeline。用淡黄色高亮“ToolSearch”和“Deferred Loading”。使用拼图、云端接口、放大镜、折叠文档、传送带等图标。

3. 正向图片提示词
画一张文章内技术解释图，主题是 Skills、MCP、ToolSearch 和 Deferred Loading 在动态能力暴露中的分层职责。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘分层架构图，清晰、克制、有工程草图感。

构图：自上而下的分层架构。最上层是“能力来源”，包含两个并列小模块：Skills 用拼图和文档图标表示经验包，MCP 用云端接口和插头图标表示外部能力桥。第二层是“能力目录”，用抽屉和索引卡表示候选能力事实表。第三层是“发现与搜索”，包含 ToolSearch 放大镜和 Deferred Loading 折叠文档，二者用淡黄色高亮。第四层是“模型投影”，包含 Skill Index、Resource Handles、Visible Tools 三张小卡片。最底层是“工具管线”，用传送带、盾牌、扳手表示 validate、permission、execute、observe。

背景有很淡的电路线、节点连线和工程草图辅助线。每层只放短标签，箭头从上到下流动。底部留出一句手写中文金句位置：“能力来源可以不同，执行入口必须统一”。整体感觉像技术博客中的手绘机制图，不是产品 UI，不是正式架构图。
