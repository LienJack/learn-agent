6 个主节点：上一轮可见集合、任务新证据、策略重算、能力 Diff、事件日志、Replay 还原。使用状态卡片、放大镜、计算齿轮、对比标记、日志卷轴、回放按钮图标。用淡黄色高亮“能力 Diff”和“事件日志”。

3. 正向图片提示词
画一张文章内技术解释图，主题是动态能力集合如何被记录为 capability diff，并支持后续归因与 replay。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘状态序列图，清晰、克制、有工程草图感。

构图：从左到右的状态序列。画面包含 6 张状态卡片，并用手绘箭头连接。① 上轮集合：几张小工具卡片，表示上一轮 visible set。② 新证据：放大镜和错误日志片段，表示测试输出提到 PR 或 CI。③ 策略重算：齿轮和筛子，表示 Discovery Policy 重新计算。④ 能力 Diff：对比标记加加号减号，用淡黄色高亮，表示 added、removed、changed。⑤ 事件日志：日志卷轴和时间戳，用淡黄色高亮，表示写入 session event log。⑥ Replay：回放按钮和还原箭头，表示后续可以还原当时模型看到的能力。

背景有很淡的电路线、节点连线和工程草图辅助线。每个节点只放很短中文标签。底部留出一句手写中文金句位置：“能力为什么出现，也要成为事实”。整体感觉像技术博客中的手绘机制图，不是产品 UI，不是正式架构图。
