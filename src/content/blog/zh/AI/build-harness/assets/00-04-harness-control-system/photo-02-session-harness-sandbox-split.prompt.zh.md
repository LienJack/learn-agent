1. 图解类型
分层架构图，因为 Session、Harness、Sandbox 分别对应记录、控制和执行边界。

2. 画面元素清单
三层或三列：Session 记录任务历史，Harness 编排权限和上下文，Sandbox 执行受限动作。外部有用户目标和项目环境。高亮 Harness 和 Sandbox 边界。

3. 正向图片提示词
画一张文章内技术解释图，主题是“Session、Harness、Sandbox 不能揉成一个对象，它们负责不同边界”。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：三列协作图。左侧是 Session：时间线和事件卡片，表示记录历史和可回放。中间是 Harness：控制面板、路由箭头、权限钥匙，使用淡黄色高亮，表示调度与策略。右侧是 Sandbox：带边界框的终端和文件夹，使用淡黄色高亮，表示受限执行环境。最左有用户目标小纸条，最右有真实项目文件夹。箭头从用户目标进入 Session 和 Harness，再由 Harness 控制 Sandbox，Sandbox 的 observation 回流到 Session。

背景用淡电路线和工程草图辅助线。每列标签短。底部留出空白金句位置。整体清晰表达职责拆分。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 7 个主节点，不要生成长段中文文字。
