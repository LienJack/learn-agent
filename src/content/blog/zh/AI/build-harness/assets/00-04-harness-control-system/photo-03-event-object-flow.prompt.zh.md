1. 图解类型
横向流水线，因为 Harness 的专业性体现在每一步都被事件对象记录和传递。

2. 画面元素清单
6 个事件卡片：UserGoal、ModelStep、ToolCall、PermissionDecision、Observation、VerificationResult。高亮 PermissionDecision 和 VerificationResult。底部有一条 Session log 时间线。

3. 正向图片提示词
画一张文章内技术解释图，主题是“Harness 把 Agent 行动拆成可审计的事件对象流”。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：横向事件流水线。上方从左到右画 6 张事件卡片：① UserGoal：用户小纸条。② ModelStep：模型芯片和下一步箭头。③ ToolCall：小 JSON 卡片。④ PermissionDecision：钥匙和路障，使用淡黄色高亮。⑤ Observation：日志卷轴。⑥ VerificationResult：测试勾选，使用淡黄色高亮。每张卡片之间用手绘箭头连接。下方画一条细长 Session log 时间线，所有事件卡片向下投影到时间线，表示可回放和审计。

背景有很淡的电路线和工程辅助线。英文事件名可以保留短标签，不要长代码。底部留空白金句区域。
