1. 图解类型
环形循环图，因为四个部件的重点是相互交接并推动下一轮，而不是静态分层。

2. 画面元素清单
4 个主节点：State、Model、Loop、Tools，加 1 个 Observation 回流节点。State 和 Tools 高亮。每个节点有简单图标：状态卡片、模型芯片、循环箭头、扳手、日志卷轴。

3. 正向图片提示词
画一张文章内技术解释图，主题是“Model、Loop、Tools、State 组成最小 Agent 闭环”。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：环形闭环流程图，5 个节点按顺时针排列。① State：叠放的状态卡片，表示当前任务现场，使用淡黄色高亮。② Model：小芯片和问号，表示判断下一步。③ Loop：循环箭头和节拍器，表示推进轮次。④ Tools：扳手和终端窗口，表示接触真实世界，使用淡黄色高亮。⑤ Observation：日志卷轴和回流箭头，表示工具结果写回状态。箭头要清楚地从 Observation 回到 State。

背景有很淡的电路线和工程草图辅助线。节点标签短，底部留出一行手写金句位置。整体感觉像解释系统机制的手绘博客配图。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 5 个主节点，不要生成长段中文文字。
