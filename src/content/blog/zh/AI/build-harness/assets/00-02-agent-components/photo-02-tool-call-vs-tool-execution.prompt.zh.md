1. 图解类型
横向流水线，因为这里要区分“模型生成意图”和“系统执行动作”的不同阶段。

2. 画面元素清单
6 个节点：模型判断、tool call、协议校验、权限检查、工具执行、observation 回填。高亮 tool call 与工具执行之间的边界。用一条竖向分界线标出模型侧和系统侧。

3. 正向图片提示词
画一张文章内技术解释图，主题是“tool call 不是工具执行，模型只提出意图，系统负责落地”。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：横向流水线。画面中间有一条竖向虚线分界，左侧标为“模型侧”，右侧标为“系统侧”。从左到右画 6 个节点：① 判断：模型芯片。② tool call：小 JSON 卡片，使用淡黄色高亮。③ 协议校验：筛网图标。④ 权限：钥匙和门禁。⑤ 执行：扳手和终端，使用淡黄色高亮。⑥ observation：日志卷轴回流到模型上下文。箭头穿过分界线时画得更粗，表示责任边界切换。

背景保持淡电路线和草图辅助线。节点标签短，不画真实代码。底部留出一句空白金句位置。整体像工程边界解释图。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 6 个主节点，不要生成长段中文文字。
