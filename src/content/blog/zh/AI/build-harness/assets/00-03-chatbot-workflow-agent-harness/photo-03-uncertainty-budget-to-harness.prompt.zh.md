1. 图解类型
输入-处理-输出图，因为它要表达不确定性和风险如何转化为 Harness 需求。

2. 画面元素清单
输入侧 3 个压力：路径不确定、环境会变、动作有风险。处理中间是 Agent 动态决策。输出侧是 Harness 三件事：权限、观测、恢复。高亮风险压力和 Harness。

3. 正向图片提示词
画一张文章内技术解释图，主题是“不确定性预算越高，越需要 Harness 承接控制成本”。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：输入-处理-输出布局。左侧画 3 张压力卡片：① 路径不确定：分叉路标。② 环境会变：文件夹旁有小闪电。③ 动作有风险：警示三角，使用淡黄色高亮。中间画 Agent 动态决策核心：模型芯片和循环箭头。右侧画 Harness 控制面板，使用淡黄色高亮，面板下有 3 个短节点：权限、观测、恢复，分别用钥匙、日志卷轴、回放箭头表示。箭头从压力流向 Agent，再流向 Harness。

背景有淡电路线和草图辅助线。节点标签短。底部留空白金句位置。整体不要像商业漏斗图，要像工程机制草图。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 7 个主节点，不要生成长段中文文字。
