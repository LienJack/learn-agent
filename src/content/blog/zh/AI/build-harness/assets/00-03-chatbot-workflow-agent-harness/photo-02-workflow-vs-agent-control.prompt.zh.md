1. 图解类型
横向流水线，因为这里要比较 Workflow 和 Agent 的控制权差异，并用左右分栏表达控制权位置不同。

2. 画面元素清单
左侧 Workflow：固定步骤轨道，代码控制下一步。右侧 Agent：观察、判断、行动的动态循环，模型参与选择下一步。中间用竖线分隔。高亮两边的“控制权”位置。

3. 正向图片提示词
画一张文章内技术解释图，主题是“Workflow 和 Agent 的区别不是智能程度，而是控制权放在哪里”。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：左右分栏对照。左侧标题短标签“Workflow”，画一条固定轨道：拉代码、跑测试、发日志三个小节点顺序连接，上方有一只代码夹板写着“代码控制”，使用淡黄色高亮。右侧标题短标签“Agent”，画一个小循环：观察、判断、行动、反馈，循环中“判断”节点由模型芯片表示并用淡黄色高亮。中间竖向虚线分隔，两边箭头风格不同：左侧笔直，右侧有分叉和回路。

背景用很淡的工程辅助线。避免长中文句子。底部留空白金句区域。整体像技术文章里的控制边界对照图。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 8 个主节点，不要生成长段中文文字。
