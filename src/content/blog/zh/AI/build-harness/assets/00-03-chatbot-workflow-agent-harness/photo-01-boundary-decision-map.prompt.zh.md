1. 图解类型
decision path + layered architecture，主图表现“从需求不确定性判断系统形态”的决策路径，并在右侧补一个小型 Harness 分层外壳。

2. 画面元素清单
- 左侧入口：用户需求，图标用对话气泡。
- 第一分支：只需对话，节点写 ChatBot，图标用聊天窗口。
- 第二分支：流程确定，节点写 Workflow，图标用流程箭头。
- 第三分支：动态决策，节点写 Agent，图标用循环箭头和扳手。
- Agent 进入真实环境后，右侧出现 Harness 外部控制面，图标用外壳、盾牌、日志卡片、验证勾选。
- 高亮元素：Agent 到 Harness 的箭头，用淡黄色标出“复杂度成本 / 控制责任”。
- 底部注释：不是升级路线，而是边界选择。

3. 正向图片提示词
米白色纸张背景，轻微纸张纹理，黑色手绘签字笔线稿，一张中文技术解释图。左侧是“用户需求”入口，向右分成三条主要判断路径：ChatBot、Workflow、Agent。ChatBot 用聊天窗口图标，Workflow 用固定流程箭头图标，Agent 用循环箭头和扳手图标。Agent 节点右侧连接一个 Harness 外部控制面，用外壳、盾牌、日志卡片、验证勾选表示权限、状态、观测、恢复和验证。用淡黄色高亮 Agent 到 Harness 的箭头，标注“复杂度成本 / 控制责任”。整体像工程白板草图，线条略不均，节点标签简短清晰，留白充足。

4. 负向提示词
不要 3D 渲染，不要赛博朋克，不要复杂彩色渐变，不要照片风，不要密集小字，不要英文长句，不要真实人物，不要公司 logo，不要科技感蓝紫背景，不要把四个概念画成从低到高的等级阶梯。
