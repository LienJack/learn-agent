1. 图解类型

Decision path。重点是把本地工具调用按动作语义分到不同风险等级，并导向 allow、ask 或 deny。

2. 画面元素清单

一个入口节点“Tool Intent”；中间分叉为 Read、Search、Edit/Write、Bash；右侧是风险等级 R1 到 R5；末端是 allow、ask、sandbox、deny；高亮 R4 高风险执行和 R5 禁止动作；背景保持工程草图感。

3. 正向图片提示词

画一张文章内技术解释图，主题是 Local Tool Bundle 的风险分级决策路径。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：决策路径图。从左侧一个入口节点“Tool Intent”开始，手绘箭头进入中央分叉路标。分叉路标有四条路径：① Read：文件图标，指向“R1 只读”；② Search：放大镜图标，指向“R1 搜索”；③ Edit/Write：铅笔和文件图标，指向“R2 写入”；④ Bash：终端图标，先进入“命令分类”，再分到“R3 执行”“R4 高风险”“R5 禁止”。右侧末端画三个小门：allow、ask、deny，R3 可连到 sandbox 小盒子。

高亮：用淡黄色强调“R4 高风险”和“R5 禁止”两个节点。

背景：很淡的电路线、节点连线和工程草图辅助线，不要抢主体。

文字：节点标签短；底部留出一行手写金句位置：“风险来自动作语义，不只来自工具名”。如果中文文字不稳定，则只留空白区域给后期添加。

整体感觉像技术博客中的手绘机制图，不是产品 UI，不是正式架构图。元素清晰，层次明确，箭头关系明确。

4. 负向提示词

不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 8 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
