1. 图解类型

Layered architecture。重点是解释 Local Tool Bundle 不是函数列表，而是夹在模型意图和本机环境之间的受控运行时层。

2. 画面元素清单

五层纵向结构：模型意图、工具契约、校验与权限、本地执行器、文件系统与终端；右侧有一条细审计链；高亮“工具契约”和“权限运行时”；背景是淡电路线；底部留一句短 caption。

3. 正向图片提示词

画一张文章内技术解释图，主题是 Local Tool Bundle 作为模型意图进入本机环境前的受控运行时层。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：分层架构图，自上而下五层。最上层是“模型意图”，图标是对话气泡和小齿轮；第二层是“工具契约”，图标是协议卡片和 JSON 括号，用淡黄色高亮；第三层是“校验/权限”，图标是漏斗和盾牌，用淡黄色高亮；第四层是“本地执行器”，图标是扳手和终端窗口；第五层是“文件/终端”，图标是文件夹、放大镜、命令行窗口。右侧画一条细竖向审计链，节点短标为“提议”“决策”“执行”“观察”。各层之间用手绘箭头连接，表现模型不能直接碰本机环境，必须穿过运行时。

高亮：用淡黄色强调“工具契约”和“校验/权限”两层。

背景：很淡的电路线、节点连线和工程草图辅助线，不要抢主体。

文字：节点标签短；底部留出一行手写金句位置：“工具不是函数，是受控能力”。如果中文文字不稳定，则只留空白区域给后期添加。

整体感觉像技术博客中的手绘机制图，不是产品 UI，不是正式架构图。元素清晰，层次明确，箭头关系明确。

4. 负向提示词

不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 8 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
