1. 图解类型
state sequence，因为这张图要表现 streaming tool-call delta 从碎片状态逐步拼接成完整 ToolIntent 的时间顺序，并强调执行不能抢跑。

2. 画面元素清单
6 个状态节点：text delta、tool start、args part 1、args part 2、complete intent、tool pipeline；在前四个状态下方画“不能执行”的小锁；淡黄色高亮 complete intent 和 tool pipeline；用时间轴箭头连接；底部留短 caption。

3. 正向图片提示词
画一张文章内技术解释图，主题是 streaming tool-call delta 必须先拼接成完整 ToolIntent，不能边流边执行。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘状态序列图，清晰、克制、有工程草图感。

构图：从左到右的状态时间轴。画面包含 6 个关键状态节点，并用手绘箭头连接：① text delta：小 token 点流图标，标签“文本”。② tool start：工具卡片打开图标，标签“开始”。③ args part 1：半张 JSON 纸片图标，标签“参数片段”。④ args part 2：另一半 JSON 纸片图标，标签“继续拼接”。⑤ complete intent：完整工具意图卡片图标，标签“完整意图”，用淡黄色高亮。⑥ tool pipeline：扳手和盾牌图标，标签“执行管线”，用淡黄色高亮。

在前四个状态下方画小锁或暂停符号，表示“不能执行”；只有完整意图之后，箭头才进入执行管线。可以在时间轴上方画一条细线表示 streaming 正在流动。

背景：很淡的电路线、节点连线和工程草图辅助线，不要抢主体。

文字：节点标签短；底部可留出一行手写金句位置：“事件可以流动，执行不能抢跑”。如果中文文字不稳定，则只留空白区域给后期添加。

整体感觉像技术博客中的手绘机制图，不是产品 UI，不是正式架构图。元素清晰，层次明确，箭头关系明确。

4. 负向提示词
不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 8 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
