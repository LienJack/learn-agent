1. 图解类型

circular loop，因为这张图要说明最小 Agent Loop 如何从单次回答变成围绕真实观察不断推进的闭环。

2. 画面元素清单

中央是“小型 CLI Agent”控制核心；环形节点包含用户目标、Think、Act、Tool、Observe、Final；用淡黄色高亮 Observe 回到 Think 的箭头和 Act 节点；右侧放一个小型项目文件夹与测试失败标记；底部留一句金句位置。

3. 正向图片提示词

画一张文章内技术解释图，主题是最小 Agent Loop 如何把单次回答变成多步行动。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：环形流程图。画面中央是一个简洁的 CLI Agent 控制核心，像小终端窗口加循环箭头。围绕中心画 6 个关键节点，并用顺时针手绘箭头连接：① 用户目标：对话气泡 + 小旗，表示“修复测试失败”。② Think：芯片 + 放大镜，表示判断缺少什么事实。③ Act：扳手 + 结构化卡片，用淡黄色高亮，表示模型提出工具意图。④ Tool：文件夹 + 终端，表示读取文件或运行测试。⑤ Observe：日志纸条 + 眼睛，用淡黄色高亮，表示结果回填。⑥ Final：勾选标记 + 简短报告，表示最终结论。

高亮：用淡黄色强调 Act、Observe，以及 Observe 回到 Think 的箭头。

背景：很淡的电路线、节点连线和工程草图辅助线，不要抢主体。

文字：节点标签短；底部可留出一行手写金句位置：“观察结果改变下一轮判断”。如果中文文字不稳定，则只留空白区域给后期添加。

整体感觉像技术博客中的手绘机制图，不是产品 UI，不是正式架构图。元素清晰，层次明确，箭头关系明确。

4. 负向提示词

不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 8 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
