1. 图解类型

state sequence，因为这张图要把 Agent Loop 解释成有状态转移和停止边界的运行时，而不是无脑 while true。

2. 画面元素清单

横向状态卡片：Ready、Thinking、Acting、Observing、Finished、Stopped；每张卡片配简单图标；预算计数器和停止牌作为高亮元素；箭头展示 Thinking 到 Acting 或 Finished 的分叉，以及 Observing 回到 Ready。

3. 正向图片提示词

画一张文章内技术解释图，主题是最小 Agent Loop 的状态机与停止边界。

画风：米白色纸张背景，黑色手绘签字笔线稿，线条粗细略不均，少量淡黄色高亮，editorial technical illustration，技术博客手绘流程图，清晰、克制、有工程草图感。

构图：状态序列图。画面从左到右排列 6 张状态卡片，并用手绘箭头连接：① Ready：状态账本图标，表示当前现场准备好。② Thinking：芯片 + 思考气泡，表示模型判断下一步。③ Acting：扳手 + 工具卡，表示执行受控工具。④ Observing：日志纸 + 眼睛，表示整理工具结果。⑤ Finished：小旗 + 勾选，表示自然结束。⑥ Stopped：停止牌 + 计数器，用淡黄色高亮，表示预算、轮次、中断或错误导致停止。

高亮：用淡黄色强调 Stopped 状态、预算计数器，以及 Observing 回到 Ready 的箭头。

背景：很淡的电路线、节点连线和工程草图辅助线，不要抢主体。

文字：节点标签短；底部可留出一行手写金句位置：“能继续，也要能停下”。如果中文文字不稳定，则只留空白区域给后期添加。

整体感觉像技术博客中的手绘机制图，不是产品 UI，不是正式架构图。元素清晰，层次明确，箭头关系明确。

4. 负向提示词

不要照片写实，不要 3D，不要复杂 UI 截图，不要大量代码，不要密集小字，不要复杂表格，不要赛博朋克，不要霓虹色，不要深色背景，不要彩色卡通，不要复杂阴影，不要过多装饰，不要超过 8 个主节点，不要让背景电路线比主体更明显，不要生成长段中文文字。
