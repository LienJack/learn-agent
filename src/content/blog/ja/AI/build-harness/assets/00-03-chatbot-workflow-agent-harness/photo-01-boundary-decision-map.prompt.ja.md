1. 図解タイプ
decision path + layered architecture。主図では「要求の不確実性からシステム形態を判断する」決定経路を表し、右側に小型の Harness 階層外殻を補足する。

2. 画面要素リスト
- 左側入口：ユーザー要求。アイコンは会話吹き出し。
- 第一分岐：対話だけでよい。ノードは ChatBot、アイコンはチャットウィンドウ。
- 第二分岐：フローが確定している。ノードは Workflow、アイコンはフロー矢印。
- 第三分岐：動的判断。ノードは Agent、アイコンはループ矢印とレンチ。
- Agent が実環境に入った後、右側に Harness 外部制御面を置く。外殻、盾、ログカード、検証チェックで権限、状態、観測、復旧、検証を表す。
- ハイライト要素：Agent から Harness への矢印を淡い黄色で示し、「複雑度コスト / 制御責任」を表す。
- 下部注釈：アップグレード経路ではなく、境界選択。

3. 正向画像提示詞
アイボリーの紙背景、軽い紙の質感、黒い手描きサインペンの線画、日本語の技術解説図。左側に「ユーザー要求」の入口を置き、右へ向けて ChatBot、Workflow、Agent の 3 つの主要判断経路に分ける。ChatBot はチャットウィンドウのアイコン、Workflow は固定フロー矢印のアイコン、Agent はループ矢印とレンチのアイコンを使う。Agent ノードの右側に Harness 外部制御面を接続し、外殻、盾、ログカード、検証チェックで権限、状態、観測、復旧、検証を示す。Agent から Harness への矢印を淡い黄色で強調し、「複雑度コスト / 制御責任」と短くラベルする。全体はエンジニアリング白板スケッチのようにし、線は少し不均一、ノードラベルは短く明快、余白は十分に残す。

4. 負向提示詞
3D レンダリングにしない、サイバーパンクにしない、複雑なカラーグラデーションにしない、写真風にしない、密集した小さな文字を入れない、長い英語文を入れない、実在人物を入れない、会社ロゴを入れない、青紫のテック背景にしない、4 つの概念を低いものから高いものへの階段として描かない。
