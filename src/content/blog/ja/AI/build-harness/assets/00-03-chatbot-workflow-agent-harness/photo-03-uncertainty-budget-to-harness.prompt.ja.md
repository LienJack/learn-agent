1. 図解タイプ
入力-処理-出力図。不確実性とリスクがどのように Harness 需要へ変わるかを表すため。

2. 画面要素リスト
入力側に 3 つの圧力：経路が不確定、環境が変化する、行動にリスクがある。処理の中央は Agent の動的意思決定。出力側は Harness の 3 要素：権限、観測、復旧。リスク圧力と Harness を強調する。

3. ポジティブ画像プロンプト
「不確実性予算が高いほど、Harness が制御コストを引き受ける必要がある」というテーマの、記事内向け技術解説図を描く。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線幅は少し不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向けの手描きフローチャート、明快で抑制され、エンジニアリングスケッチ感がある。

構図：入力-処理-出力レイアウト。左側に 3 枚の圧力カードを描く：① 経路不確定：分岐標識。② 環境変化：フォルダの横に小さな稲妻。③ 行動リスク：警告三角、淡い黄色で強調する。中央に Agent 動的意思決定コアを描く：モデルチップとループ矢印。右側に Harness 制御パネルを描き、淡い黄色で強調する。パネル下に 3 つの短いノード：権限、観測、復旧を置き、それぞれ鍵、ログ巻物、リプレイ矢印で表す。矢印は圧力から Agent へ、さらに Harness へ流れる。

背景には淡い回路線とスケッチ補助線を置く。ノードラベルは短くする。下部に空白の名言スペースを残す。全体は商業的なファネル図ではなく、エンジニアリングメカニズムのスケッチにする。
