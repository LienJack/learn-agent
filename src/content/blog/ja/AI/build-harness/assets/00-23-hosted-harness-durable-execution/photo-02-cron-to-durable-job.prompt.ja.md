1. 図解タイプ

Circular loop。この図は、Cron が一回限りのコマンドではなく、job、lease、session、resume、notification を中心に継続的に進む復元可能なライフサイクルであることを表すため。

2. 画面要素リスト

画面は時計回りのループで、7 個のノード：Cron、Job、Queue、Worker、Session、Pause/Resume、Notify。中央に小型 Agent タスクカードを置く。Cron はカレンダー時計アイコン、Job はタスクカード、Queue はキュー線、Worker は小サーバー、Session はイベントログ帳で淡い黄色に強調、Pause/Resume は一時停止と再生ボタン、Notify はメッセージ吹き出しで淡い黄色に強調。矢印は閉ループを作り、タスクが完了する場合も、一時停止後に再開する場合もあることを示す。背景にごく薄い回路線と破線 checkpoint。

3. ポジティブ画像プロンプト

記事内の技術解説図を描く。テーマは Cron が「定時コマンド実行」から「復元可能な durable job lifecycle の作成」へアップグレードされる仕組み。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線の太さは少し不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向け手描きフロー図、明快で抑制され、エンジニアリングスケッチ感がある。

構図：環状フロー図。中央に小型 Agent タスクカードを置き、短いラベル「Agent Job」を書く。中心の周囲に時計回りで 7 個のフローノードを置き、手描き矢印で閉ループに接続する：① Cron：カレンダー + 小時計、定時トリガーを表す。② Job：タスクカード、冪等なタスク作成を表す。③ Queue：待ち行列と小点、永続化された待機を表す。④ Worker：小サーバー、lease を取得することを表す。⑤ Session：イベント台帳、事実ソースを記録することを表し、淡い黄色で強調。⑥ Pause/Resume：一時停止キーと再生キー、承認待ちまたは再開を表す。⑦ Notify：メッセージ吹き出し、ユーザー通知を表し、これも淡い黄色で強調。

矢印は、Cron が直接コマンドを走らせるのではなく job を作ること、worker は一時的な推進者にすぎないこと、session が事実を保存すること、タスクが一時停止、再開、完了できることを表す。ノードラベルは短く、下部に手書きの日本語フレーズ用スペースを残す：「Cron はタスクを作り、世界を直接実行しない」。全体は技術ブログ内の手描きメカニズム図の雰囲気で、製品 UI でも正式なアーキテクチャ図でもない。

4. ネガティブプロンプト

写真写実にしない、3D にしない、複雑な UI スクリーンショットにしない、大量のコードを入れない、密集した小文字を入れない、複雑な表にしない、サイバーパンクにしない、ネオン色にしない、暗い背景にしない、カラフルな漫画調にしない、複雑な影を入れない、装飾を増やしすぎない、主要ノードは 8 個を超えない、背景の回路線を主体より目立たせない、長い日本語文章を生成しない。
