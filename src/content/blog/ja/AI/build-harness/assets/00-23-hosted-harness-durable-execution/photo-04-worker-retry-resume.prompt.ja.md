1. 図解タイプ

State sequence。この図は、worker がクラッシュしてもタスクが消えるわけではなく、キュー再割り当て、session replay、artifact 検証、sandbox 再構築によって継続できることを示すため。

2. 画面要素リスト

画面は左から右への状態シーケンスで、7 個のノード：Job Queue、Worker A、Crash、Session Replay、Artifact Check、Worker B、Resume。Job Queue はキューと lease カードアイコン、Worker A は小サーバー、Crash は停電の稲妻、Session Replay はイベントログ再生アイコンで淡い黄色に強調、Artifact Check はフォルダと検証チェック、Worker B は別の小サーバー、Resume は再生ボタンと前進矢印で淡い黄色に強調。背景は破線 checkpoint と薄い回路線。下部にフレーズ用スペース：「worker は死んでも、事実は失われてはいけない」。

3. 正向き画像プロンプト

記事内の技術解説図を描く。テーマは Hosted Harness が remote worker のクラッシュ後、session と artifact によって Agent の長時間タスクを復旧する仕組み。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線の太さは少し不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向け手描きフロー図、明快で抑制され、エンジニアリングスケッチ感がある。

構図：状態シーケンス図。左から右へ 7 個のノードを並べ、手描き矢印で接続する：① Job Queue：キュー線と lease カード、タスクを再割り当てできることを表す。② Worker A：小サーバー、最初の実行者を表す。③ Crash：停電の稲妻と切れた線、worker クラッシュを表す。④ Session Replay：イベントログ帳と再生矢印、事実ソースから復旧することを表し、淡い黄色で強調。⑤ Artifact Check：フォルダと検証チェック、ログ、patch、テスト結果の証拠を確認することを表す。⑥ Worker B：別の小サーバー、新しい worker が引き継ぐことを表す。⑦ Resume：再生ボタンと前進矢印、長時間タスクを続行することを表し、淡い黄色で強調。

矢印は、worker クラッシュ後に job がキューへ戻り、新しい worker は最初からやり直すのではなく、session を replay し、artifact を検証し、sandbox を再構築してから続行することを表す。ノードラベルは短く、下部に手書きの日本語フレーズ用スペースを残す：「worker は死んでも、事実は失われてはいけない」。全体は技術ブログ内の手描きメカニズム図の雰囲気で、製品 UI でも正式な障害トポロジ図でもない。

4. ネガティブプロンプト

写真写実にしない、3D にしない、複雑な UI スクリーンショットにしない、大量のコードを入れない、密集した小文字を入れない、複雑な表にしない、サイバーパンクにしない、ネオン色にしない、暗い背景にしない、カラフルな漫画調にしない、複雑な影を入れない、装飾を増やしすぎない、主要ノードは 8 個を超えない、背景の回路線を主体より目立たせない、長い日本語文章を生成しない。
