1. 図解タイプ
horizontal pipeline。この図は、候補能力がどのように段階的にフィルタリングされ、このターンでモデルに見える最小集合になり、最終的に統一されたツール実行パイプラインへ入るかを説明するため。

2. 画面要素リスト
6 個の主要ノード：能力ソース、能力カタログ、発見ポリシー、軽量インデックス、可視集合、ツールパイプライン。アイコンはそれぞれツールボックス、カタログカード、ふるい、インデックスカード、目、コンベヤー。淡い黄色で「発見ポリシー」と「可視集合」を強調。背景にはごく薄い回路線とエンジニアリングスケッチの補助線を入れる。下部に短い一文の位置を残す：「存在することと見えることは違い、見えることと実行できることも違う」。

3. 正向き画像プロンプト
記事内の技術解説図を描く。テーマは Capability Discovery が大量の候補能力を、このターンの最小可視集合へ絞り込む仕組み。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線の太さは少し不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向け手描きフロー図、明快で抑制され、エンジニアリングスケッチ感がある。

構図：左から右への水平パイプライン。画面には 6 個の重要ノードを置き、手描き矢印で接続する。① 能力ソース：ツールボックスのアイコン、横に小さく「Tools / Skills / MCP」。② 能力カタログ：カタログカードと引き出しアイコン、候補能力のファクトテーブルを表す。③ 発見ポリシー：ふるいアイコン、淡い黄色で強調し、タスク、パス、権限、予算でフィルタリングすることを表す。④ 軽量インデックス：数枚の小さなインデックスカード、名称と短い説明だけを公開することを表す。⑤ 可視集合：目と小さなメニューアイコン、淡い黄色で強調し、このターンでモデルが見られる能力を表す。⑥ ツールパイプライン：コンベヤーとレンチのアイコン、すべての実行可能アクションが validate / permission / execute / observe に入ることを表す。

背景にはごく薄い回路線、ノード接続線、エンジニアリングスケッチ補助線を入れるが、主題より目立たせない。各ノードには短い日本語ラベルのみを置く。下部に手書きの日本語フレーズ用スペースを残す：「存在することと見えることは違い、見えることと実行できることも違う」。全体は技術ブログ内の手描きメカニズム図の雰囲気で、製品 UI でも正式なアーキテクチャ図でもない。
