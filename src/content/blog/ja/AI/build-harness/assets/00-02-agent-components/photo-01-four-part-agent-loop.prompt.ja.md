1. 図解タイプ
円形ループ図。4 つの部品の要点は、静的な階層ではなく、互いに受け渡しながら次のターンを進めることにあるため。

2. 画面要素リスト
4 つの主要ノード：State、Model、Loop、Tools に、1 つの Observation 回流ノードを加える。State と Tools を強調する。各ノードに簡単なアイコンを付ける：状態カード、モデルチップ、ループ矢印、レンチ、ログ巻物。

3. ポジティブ画像プロンプト
「Model、Loop、Tools、State が最小 Agent 閉ループを構成する」というテーマの、記事内向け技術解説図を描く。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線幅は少し不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向けの手描きフローチャート、明快で抑制され、エンジニアリングスケッチ感がある。

構図：円形の閉ループフロー図。5 つのノードを時計回りに配置する。① State：重なった状態カード。現在のタスク現場を示し、淡い黄色で強調する。② Model：小さなチップと疑問符。次の一手を判断することを示す。③ Loop：ループ矢印とメトロノーム。ターンを進めることを示す。④ Tools：レンチと端末ウィンドウ。現実世界に触れることを示し、淡い黄色で強調する。⑤ Observation：ログ巻物と回流矢印。ツール結果が状態へ書き戻されることを示す。Observation から State へ戻る矢印を明確に描く。

背景にはごく淡い回路線とエンジニアリングスケッチの補助線を置く。ノードラベルは短くし、下部に一行の手書き名言スペースを残す。全体はシステムメカニズムを説明する手描きブログ挿絵のようにする。
