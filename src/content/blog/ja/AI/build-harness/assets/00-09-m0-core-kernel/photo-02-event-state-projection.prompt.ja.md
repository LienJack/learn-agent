1. 図解タイプ
横方向パイプライン。イベントという事実ソースが state に折りたたまれ、さらに今回のモデル入力へ投影される流れを説明する。

2. 画面要素リスト
5 つの主ノード：Event Log、State Reducer、Conversation State、Context Projection、ModelRequest。横に小さな Trace / Replay ノードを添える。Event Log と Context Projection を強調する。アイコンはログ巻物、漏斗、状態カード、投影ライト、リクエスト封筒、虫眼鏡。

3. 正向き画像プロンプト
記事内の技術解説図を描く。テーマは「Conversation State は事実ログの投影であり、事実そのものではない」。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線幅はわずかに不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向けの手描きフロー図、明瞭で抑制され、エンジニアリングのラフスケッチ感がある。

構図：左から右への横方向パイプライン。① Event Log：ログ紙の巻物と時間目盛りで事実ソースを表し、淡い黄色で強調する。② State Reducer：漏斗と折りたたみ矢印で、イベントから現在の現場を折りたたむことを表す。③ Conversation State：複数の状態カードで pending intent、usage、status、messages を表す。④ Context Projection：投影ライトがモデル作業台を照らし、淡い黄色で強調し、今回必要な情報だけを選ぶことを示す。⑤ ModelRequest：封筒またはリクエスト包みで、provider に送る messages + tools を表す。右下に小さな Trace / Replay の虫眼鏡を描き、Event Log から破線を伸ばす。

背景にはごく薄い回路線、ノード接続線、エンジニアリングスケッチの補助線を入れるが、主題より目立たせない。 各ノードには短いラベルのみを置く。下部には後から文字を追加できる空白の一行を残す。全体は技術ブログ内の手描きメカニズム図であり、製品 UI や正式なアーキテクチャ図ではない。

4. ネガティブプロンプト
写真写実、3D、複雑な UI スクリーンショット、大量のコード、密集した小さな文字、複雑な表、サイバーパンク、ネオン色、暗い背景、カラフルな漫画調、複雑な影、過剰な装飾、6 個を超える主ノード、背景の回路線が主体より目立つ表現、長い日本語文章の生成は避ける。
