1. 図解タイプ
決定経路図。候補内容がコンテキストに入る前に複数の検査ゲートを通ることを示す。

2. 画面要素リスト
左側に候補内容カード。順に 5 つの gate：関連性、現在の事実、権限、信頼、予算。各 gate に許可/除外/要約の分岐。信頼と予算 gate を強調し、右側に Model Input を出力する。

3. 正向き画像プロンプト
記事内の技術解説図を描く。テーマは「関連しているだけではコンテキストに入れず、候補内容は Context Policy の複数の門を通る必要がある」。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線幅はわずかに不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向けの手描きフロー図、明瞭で抑制され、エンジニアリングのラフスケッチ感がある。

構図：左から右への決定経路。左側に候補内容カードの束、ラベル Candidate。中央に 5 つの小ゲートを連続して描く：① Relevance は虫眼鏡。② Fresh Fact は時計と更新矢印。③ Permission は鍵。④ Trust は盾で淡い黄色ハイライト。⑤ Budget は token 計量尺で淡い黄色ハイライト。各 gate の下に Exclude / Summarize / Cite へ向かう細い分岐を置き、長文は書かない。右側に整理された Model Input カードを描く。

背景にはごく薄い回路線、ノード接続線、エンジニアリングスケッチの補助線を入れるが、主題より目立たせない。

文字：ノードラベルは短い日本語または英語のみにする。下部には後から金句を追加できる空白の一行を残す。全体は技術ブログ中の手描きメカニズム図であり、製品 UI や正式なアーキテクチャ図ではない。要素は明瞭で、階層と矢印関係をはっきりさせる。

4. ネガティブプロンプト
