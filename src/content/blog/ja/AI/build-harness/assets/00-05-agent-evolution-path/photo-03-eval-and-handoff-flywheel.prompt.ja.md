1. 図解タイプ
円形フライホイール図。復旧可能、評価可能、委譲可能という性質が Managed Agent の継続的改善ループを形成するため。

2. 画面要素リスト
6 つのノード：Session log、Replay、Eval、Harness Fix、Sub-agent Handoff、Production Run。Eval と Harness Fix を強調する。中心に Managed Agent 制御コアを書く。矢印で時計回りのフライホイールを形成する。

3. ポジティブ画像プロンプト
「Managed Agent はリプレイ、評価、委譲によって改善フライホイールを形成する」というテーマの、記事内向け技術解説図を描く。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線幅は少し不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向けの手描きフローチャート、明快で抑制され、エンジニアリングスケッチ感がある。

構図：円形フライホイール図。中央に Managed Agent 制御コアを描く：小さな制御台とガードレール。外周に時計回りで 6 つのノードを配置する：① Session log：イベントタイムライン。② Replay：リプレイ矢印。③ Eval：テスト定規とチェック、淡い黄色で強調する。④ Harness Fix：レンチで制御台を修正する、淡い黄色で強調する。⑤ Sub-agent Handoff：タスクカードの引き継ぎ。⑥ Production Run：プロジェクトフォルダと実行インジケータ。ノード間を太さが少し不均一な手描き矢印で結び、継続ループにする。

背景にはごく淡い回路線とエンジニアリングスケッチ補助線を置く。ラベルは短くし、下部に一行の空白名言スペースを残す。全体は運用改善メカニズムを説明する技術ブログの手描き図のようにする。
