1. 図解タイプ
状態シーケンス図。この図は Agent の責任が、回答できる段階から托管実行できる段階へ徐々に進化することを示すため。

2. 画面要素リスト
5 つの段階：Chat Agent、Tool Agent、Runtime Agent、Managed Agent、Harness。各段階に新しく増える責任の小アイコンを付ける。Runtime Agent と Managed Agent を強調する。これらは demo から実システムへ進む転換点だから。

3. ポジティブ画像プロンプト
「Agent が Chat Agent から Harness へ自然に進化する」というテーマの、記事内向け技術解説図を描く。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線幅は少し不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向けの手描きフローチャート、明快で抑制され、エンジニアリングスケッチ感がある。

構図：左から右への状態シーケンス。① Chat Agent：会話吹き出しと messages カード。② Tool Agent：レンチと tool schema カード。③ Runtime Agent：時計、予算バー、リプレイ矢印、淡い黄色で強調する。④ Managed Agent：サンドボックス枠、権限キー、trace 線、淡い黄色で強調する。⑤ Harness：外部制御台とガードレール。モデル外部の制御システムを示す。各段階の下には 1-2 個の短いラベルだけを置く。矢印で、責任が現実の圧力によって一歩ずつ押し出されることを示す。

背景には淡い回路線とエンジニアリングスケッチ補助線を置く。下部に空白の名言スペースを残す。全体は製品ロードマップではなく、エンジニアリング進化メカニズム図にする。
