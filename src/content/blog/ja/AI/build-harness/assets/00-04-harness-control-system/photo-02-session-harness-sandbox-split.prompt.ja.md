1. 図解タイプ
階層アーキテクチャ図。Session、Harness、Sandbox はそれぞれ記録、制御、実行境界に対応するため。

2. 画面要素リスト
3 層または 3 列：Session はタスク履歴を記録し、Harness は権限とコンテキストを編成し、Sandbox は制限された動作を実行する。外部にユーザー目標とプロジェクト環境を置く。Harness と Sandbox の境界を強調する。

3. ポジティブ画像プロンプト
「Session、Harness、Sandbox は 1 つのオブジェクトに混ぜてはいけない。それぞれ異なる境界を担当する」というテーマの、記事内向け技術解説図を描く。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線幅は少し不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向けの手描きフローチャート、明快で抑制され、エンジニアリングスケッチ感がある。

構図：3 列の協作図。左側は Session：タイムラインとイベントカードで、履歴記録とリプレイ可能性を示す。中央は Harness：制御パネル、ルーティング矢印、権限キーを描き、淡い黄色で強調して、スケジューリングとポリシーを示す。右側は Sandbox：境界フレーム付きの端末とフォルダを描き、淡い黄色で強調して、制限付き実行環境を示す。最左にユーザー目標の小さな紙片、最右に実プロジェクトフォルダを置く。矢印はユーザー目標から Session と Harness へ入り、Harness が Sandbox を制御し、Sandbox の observation が Session へ回流する。

背景には淡い回路線とエンジニアリングスケッチ補助線を使う。各列のラベルは短くする。下部に空白の名言スペースを残す。全体は責任分割を明確に表現する。

4. ネガティブプロンプト
写真写実にしない、3D にしない、複雑な UI スクリーンショットにしない、大量のコードを入れない、密集した小さな文字を入れない、複雑な表を入れない、サイバーパンクにしない、ネオン色にしない、暗い背景にしない、カラフルな漫画調にしない、複雑な影を入れない、装飾を増やしすぎない、主要ノードを 7 個より多くしない、長い日本語文を生成しない。
