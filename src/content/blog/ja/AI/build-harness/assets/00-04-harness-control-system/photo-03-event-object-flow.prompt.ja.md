1. 図解タイプ
横方向パイプライン。Harness の専門性は、各ステップがイベントオブジェクトとして記録・伝達されることに現れるため。

2. 画面要素リスト
6 つのイベントカード：UserGoal、ModelStep、ToolCall、PermissionDecision、Observation、VerificationResult。PermissionDecision と VerificationResult を強調する。下部に Session log タイムラインを置く。

3. 正向画像提示詞
「Harness は Agent の行動を監査可能なイベントオブジェクトの流れに分解する」というテーマの、記事内向け技術解説図を描く。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線幅は少し不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向けの手描きフローチャート、明快で抑制され、エンジニアリングスケッチ感がある。

構図：横方向イベントパイプライン。上部に左から右へ 6 枚のイベントカードを描く：① UserGoal：ユーザーの小さな紙片。② ModelStep：モデルチップと次ステップ矢印。③ ToolCall：小さな JSON カード。④ PermissionDecision：鍵とバリケード、淡い黄色で強調する。⑤ Observation：ログ巻物。⑥ VerificationResult：テストチェック、淡い黄色で強調する。各カードを手描き矢印で接続する。下部に細長い Session log タイムラインを描き、すべてのイベントカードが下へ投影されるようにして、リプレイ可能性と監査可能性を示す。

背景にはごく淡い回路線とエンジニアリング補助線を置く。英語のイベント名は短いラベルとして保持してよく、長いコードは入れない。下部に空白の名言スペースを残す。
