1. 図解タイプ

水平パイプライン図。要点は、extension がインストールから実際の実行へ進むまでに verify、trust、load、catalog、discovery、permission、tool runtime という複数のゲートを通ること。

2. 画面要素リスト

左から右への extension ライフサイクルパイプライン。ノード：Install、Verify、Trust、Manifest、Catalog、Visible Set、Permission Gate、Tool Runtime。淡い黄色で Trust と Permission Gate を強調。アイコン比喩：ダウンロード箱、検証印、ロックと盾、マニフェスト紙、カタログ引き出し、スポットライト、ゲート、レンチ。

3. ポジティブ画像プロンプト

記事内の技術解説図を描く。テーマは Agent CLI の extension がインストールから制御された実行へ進むライフサイクル。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線の太さは少し不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向け手描きフロー図、明快で抑制され、エンジニアリングスケッチ感がある。

構図：水平パイプライン図。画面は左から右へ 8 個のノードを並べ、手描き矢印で接続する。各ノードは小カードまたは小さな角丸ラベルのように見せるが、手描き線画感を保つ。パイプライン下にごく細い補助線を置き、extension が段階的に Harness の規律へ入ることを表す。

ノード：① Install：ダウンロード箱アイコン、ファイルが配置されることを表す。② Verify：検証印アイコン、バージョン、出典、checksum を表す。③ Trust：ロックと盾アイコン、ユーザーまたは組織ポリシーによる信頼を表す。④ Manifest：マニフェスト紙アイコン、provider、tool、skill、hook の宣言を表す。⑤ Catalog：カタログ引き出しアイコン、候補能力カタログを表す。⑥ Visible Set：スポットライトアイコン、このターンの可視能力を表す。⑦ Permission Gate：ゲートアイコン、パラメータ、リスク、承認を表す。⑧ Tool Runtime：レンチアイコン、制御された実行と observation を表す。

ハイライト：Trust と Permission Gate を淡い黄色で強調し、インストールは有効化ではなく、可視化は実行可能ではないことを表す。

背景：ごく薄い回路線、ノード接続線、エンジニアリングスケッチ補助線を入れ、主題より目立たせない。

文字：ノードラベルは短くする。下部に手書きの日本語フレーズ用スペースを残してよい：「インストールは有効化ではなく、見えることは実行可能ではない」。日本語文字が不安定なら、後入れ用の空白領域だけを残す。

全体は技術ブログ内の手描きメカニズム図の雰囲気で、製品 UI でも正式なアーキテクチャ図でもない。要素は明快で、階層と矢印関係がはっきりしている。
