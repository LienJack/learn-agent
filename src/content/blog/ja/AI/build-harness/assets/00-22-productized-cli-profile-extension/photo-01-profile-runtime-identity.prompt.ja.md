1. 図解タイプ

階層アーキテクチャ図。要点は、profile が表面的な好みではなく、ポリシー、ツール、context、モデル嗜好、出力プロトコルを組み合わせた実行アイデンティティであること。

2. 画面要素リスト

中心ノード：Profile 実行アイデンティティ。周囲の 5 層：Policy、Tool Bundle、Context Source、Provider Preference、Output Contract。下部は Harness Runtime で、profile はアイデンティティを宣言するだけで、実際の実行は runtime に戻ることを表す。淡い黄色で Profile と Harness Runtime を強調。アイコン比喩：盾は policy、ツールボックスは tool bundle、フォルダは context source、クラウドインターフェースは provider preference、イベントストリームカードは output contract、土台は runtime。

3. 正向き画像プロンプト

記事内の技術解説図を描く。テーマは Agent CLI の profile が実行アイデンティティを定義する仕組み。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線の太さは少し不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向け手描きメカニズム図、明快で抑制され、エンジニアリングスケッチ感がある。

構図：階層アーキテクチャ図。画面中央上部に大きめの手描きカードを置き、ラベルは「Profile」、横に短く「実行アイデンティティ」と書く。Profile の下に 5 個の小モジュールを横並びにし、細い矢印で下部の Harness Runtime 土台へ合流させる。

5 個のモジュール：① Policy：盾アイコン、権限とリスク境界を表す。② Tool Bundle：ツールボックスアイコン、デフォルトのツール組み合わせを表す。③ Context Source：フォルダと付箋アイコン、プロジェクト指示、Skill、記憶ソースを表す。④ Provider Preference：クラウドインターフェースと切替矢印、モデル嗜好と fallback を表す。⑤ Output Contract：イベントストリームカードアイコン、TTY、JSONL、Workbench の出力プロトコルを表す。

ハイライト：中央の Profile カードと下部の Harness Runtime 土台を淡い黄色で強調し、profile はアイデンティティを宣言し、runtime がそれを実行することを表す。

背景：ごく薄い回路線、ノード接続線、エンジニアリングスケッチ補助線を入れ、主題より目立たせない。

文字：ノードラベルは短くする。下部に手書きの日本語フレーズ用スペースを残してよい：「Profile は実行アイデンティティであって、テーマスキンではない」。日本語文字が不安定なら、後入れ用の空白領域だけを残す。

全体は技術ブログ内の手描きメカニズム図の雰囲気で、製品 UI でも正式なアーキテクチャ図でもない。要素は明快で、階層と矢印関係がはっきりしている。
