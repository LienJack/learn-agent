1. 図解タイプ

Layered architecture。要点は、event log を事実ソースとして、異なる実行体へ異なる context を投影し、最後に親 Agent join へ戻すこと。

2. 画面要素リスト

最下層は Event Log の長い土台。中間に 3 枚の投影カード：主 context、子タスク要約、レビュー context。上層に 3 つの役割：親 Agent、Sub-Agent、Reviewer。右側では構造化結果が親 Agent Join へ合流する。Event Log と Join を強調。アイコンはログ巻物、投影ライト、隔離された小区画、レビュー用虫眼鏡、合流ファネル。

3. 正向き画像プロンプト

記事内の技術解説図を描く。テーマは delegation における context isolation と結果統合。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線の太さは少し不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向け手描きアーキテクチャ図、明快で抑制され、エンジニアリングスケッチ感がある。

構図：階層アーキテクチャ図。下部に横長の Event Log 土台を置き、ログ巻物またはタイムラインのように描き、淡い黄色で軽く強調して事実ソースを表す。中間層には 3 枚の投影カードを置く：① 主 context：目標と次のステップ。② 子タスク要約：証拠、範囲、仮説。③ レビュー context：diff、計画、リスク。上層には対応する 3 つの実行体：親 Agent、Sub-Agent、Reviewer を置き、それぞれを隔離された小さな作業室のように描く。右側では合流ファネルと矢印で Sub-Agent の結果と Reviewer の意見を「親 Agent Join」に統合し、Join ノードを淡い黄色で強調する。

背景：ごく薄い回路線、ノード接続線、エンジニアリングスケッチ補助線を入れ、主題より目立たせない。ノードラベルは短く、長文を避ける。下部に手書きの日本語フレーズ用スペースを残してよい：「trace は事実を保存し、context は今必要なものだけを投影する」。全体は技術ブログ内の手描きメカニズム図の雰囲気で、製品 UI でも正式なアーキテクチャ図でもない。

4. ネガティブプロンプト

写真写実にしない、3D にしない、複雑な UI スクリーンショットにしない、大量のコードを入れない、密集した小文字を入れない、複雑な表にしない、サイバーパンクにしない、ネオン色にしない、暗い背景にしない、カラフルな漫画調にしない、複雑な影を入れない、装飾を増やしすぎない、主要ノードは 8 個を超えない、背景の回路線を主体より目立たせない、長い日本語文章を生成しない。
