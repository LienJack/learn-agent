1. 図解タイプ

Horizontal pipeline。要点は、delegation を「もう一度モデルを呼ぶ」ことではなく、制御されたツール呼び出しパイプラインとして描き直すこと。

2. 画面要素リスト

画面は左から右へ 7 個のノード：親 Agent、タスクパッケージ、Validate、Permission、制御された子 Agent、Observation、親 Agent Join。親 Agent は小さな指揮台アイコン、タスクパッケージはフォルダアイコン、Validate はチェックリスト、Permission は盾、制御された子 Agent は小さな作業台、Observation は状態カード、Join は合流矢印。淡い黄色で Permission と Join を強調。下部に短い一文の位置を残す：「分けるのは作業であって、制御権ではない」。

3. ポジティブ画像プロンプト

記事内の技術解説図を描く。テーマは Delegation Runtime が sub-agent 委譲を制御されたツール呼び出しへ変える仕組み。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線の太さは少し不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向け手描きフロー図、明快で抑制され、エンジニアリングスケッチ感がある。

構図：横方向の pipeline。画面には 7 個の重要ノードを置き、手描き矢印で左から右へ接続する：① 親 Agent：小さな指揮台と目標旗、主線の制御者を表す。② タスクパッケージ：フォルダとチェックリスト、objective、scope、contract を表す。③ Validate：チェックリスト、構造と範囲の検証を表す。④ Permission：盾とゲート、淡い黄色で強調し、ツールとリスクの承認を表す。⑤ 子 Agent：小さな作業台、隔離された実行体を表す。⑥ Observation：状態カード、構造化された結果を表す。⑦ Join：複数の矢印が親 Agent に合流し、淡い黄色で強調して親 Agent がレビューして統合することを表す。

背景：ごく薄い回路線、ノード接続線、エンジニアリングスケッチ補助線を入れ、主題より目立たせない。ノードラベルは短く、できれば「親 Agent」「タスク」「検証」「権限」「子 Agent」「観測」「Join」のみを使う。下部に手書きの日本語フレーズ用スペースを残す：「分けるのは作業であって、制御権ではない」。全体は技術ブログ内の手描きメカニズム図の雰囲気で、製品 UI でも正式なアーキテクチャ図でもない。
