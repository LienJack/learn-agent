1. 図解タイプ
Horizontal pipeline。この図は、Scoped Retrieval がタスク境界から audit snapshot へ至る完全な加工チェーンを説明し、裸の検索を監査可能な証拠パッケージへ変える各ステップを示すため。

2. 画面要素リスト
6 個の主要ノード：タスク Scope、候補リコール、境界フィルタ、タスク再ランキング、予算トリミング、Audit Snapshot。フォルダ、虫眼鏡、盾、ソートスライダー、はさみ、証拠箱のアイコンを使う。淡い黄色でタスク Scope と Audit Snapshot を強調。背景はごく薄い回路線とエンジニアリングスケッチ補助線。下部に手書きフレーズを予約：「先に境界を決め、次に証拠を渡す」。

3. 正向き画像プロンプト
記事内の技術解説図を描く。テーマは Scoped Retrieval がタスク境界から audit snapshot を生成する仕組み。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線の太さは少し不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向け手描きフロー図、明快で抑制され、エンジニアリングスケッチ感がある。

構図：左から右への横向きパイプライン。画面には 6 個の重要ノードを置き、手描き矢印で接続する。① タスク Scope：フォルダと境界枠アイコン、プロジェクト、ユーザー、権限、時間境界を表し、淡い黄色で強調。② 候補リコール：虫眼鏡が複数の小紙片を探す、BM25、ベクトル、イベントログ、memory の候補発見を表す。③ 境界フィルタ：盾とファネル、境界外候補が外に止められることを表す。④ タスク再ランキング：ソートスライダーと星印、現在タスクとの関連度で並べ替えることを表す。⑤ 予算トリミング：はさみで長い文書を短く切る、token budget と可視テキストのトリミングを表す。⑥ Audit Snapshot：証拠箱、ロック、引用ラベル、最終的に復盤可能なスナップショットへ書き込むことを表し、淡い黄色で強調。

ハイライト：「タスク Scope」と「Audit Snapshot」、および Scope から候補リコールへの最初の矢印を淡い黄色で強調。

背景：ごく薄い回路線、ノード接続線、エンジニアリングスケッチ補助線を入れ、主題より目立たせない。

文字：ノードラベルは短くする。下部に手書きの日本語フレーズ用スペースを残す：「先に境界を決め、次に証拠を渡す」。日本語文字が不安定なら、後入れ用の空白領域だけを残す。

全体は技術ブログ内の手描きメカニズム図の雰囲気で、製品 UI でも正式なアーキテクチャ図でもない。要素は明快で、階層と矢印関係がはっきりしている。
