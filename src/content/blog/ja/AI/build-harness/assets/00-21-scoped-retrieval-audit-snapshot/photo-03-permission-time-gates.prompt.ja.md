1. 図解タイプ
Layered architecture。この図は、検索結果がモデルへ入る前の多層ゲート、つまりデータソースアクセス、モデル可視性、時間境界、投影スナップショットを示すため。

2. 画面要素リスト
4 層構造：Knowledge Sources、Source Access Gate、Model Visibility Gate、Time Snapshot Gate、Model Context。ノードには知識ソース庫、アクセス盾、可視性の目、時計スナップショット、モデル作業台を含める。淡い黄色で Model Visibility Gate と Time Snapshot Gate を強調。下部に小さな audit log トラックを置いてよい。

3. ポジティブ画像プロンプト
記事内の技術解説図を描く。テーマは検索結果がモデルに入る前に権限と時間境界を通らなければならないこと。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線の太さは少し不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向け手描きフロー図、明快で抑制され、エンジニアリングスケッチ感がある。

構図：縦方向の階層アーキテクチャ図。下から上へ 5 層。最下層は「Knowledge Sources」で、データベース、フォルダ、session ログ、memory カードを描く。第 2 層は「Source Access」で、盾のアクセスゲートにより、システムがソース読み取りを許可するかを表す。第 3 層は「Model Visibility」で、目と遮蔽バーを淡い黄色で強調し、読めた内容が必ずモデルへ見せられるわけではないことを表す。第 4 層は「Time Snapshot」で、時計とカメラスナップショットを淡い黄色で強調し、当時のバージョンで証拠を固定することを表す。最上層は「Model Context」で、小さな作業台と短い証拠カードを描き、最終的にモデルへ入る可視投影を表す。右側に細い線を引いて下部の「Audit Log」台帳へ接続し、各ゲートの判断が記録されることを表す。

ハイライト：「Model Visibility」と「Time Snapshot」の 2 層を淡い黄色で強調。

背景：ごく薄い回路線、ノード接続線、エンジニアリングスケッチ補助線を入れ、主題より目立たせない。

文字：ノードラベルは短くする。下部に手書きの日本語フレーズ用スペースを残す：「読めることと、露出できることは違う」。日本語文字が不安定なら、後入れ用の空白領域だけを残す。

全体は技術ブログ内の手描きメカニズム図の雰囲気で、製品 UI でも正式なアーキテクチャ図でもない。要素は明快で、階層と矢印関係がはっきりしている。
