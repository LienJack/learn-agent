1. 図解タイプ
Decision path。ローカルツール呼び出しを行動意味ごとにリスク等級へ分類し、allow、ask、deny へ導く。

2. 画面要素リスト
入口ノード Tool Intent。中央は Read、Search、Edit/Write、Bash へ分岐。右側に R1 から R5 のリスク等級。終端は allow、ask、sandbox、deny。R4 高リスク実行と R5 禁止動作を強調する。

3. ポジティブ画像プロンプト
記事内の技術解説図を描く。テーマは「Local Tool Bundle のリスク分類決定経路」。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線幅はわずかに不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向けの手描きフロー図、明瞭で抑制され、エンジニアリングのラフスケッチ感がある。

構図：左側の Tool Intent 入口ノードから中央の分岐標識へ入る。分岐は 4 本：Read はファイルアイコンから R1 Read Only へ、Search は虫眼鏡から R1 Search へ、Edit/Write は鉛筆とファイルから R2 Write へ、Bash は端末から Command Classify へ入り、R3 Execute、R4 High Risk、R5 Forbidden へ分かれる。右端に allow、ask、deny の 3 つの小さな門を描き、R3 は sandbox 小箱へ接続できる。R4 High Risk と R5 Forbidden を淡い黄色で強調する。

背景にはごく薄い回路線、ノード接続線、エンジニアリングスケッチの補助線を入れるが、主題より目立たせない。

文字：ノードラベルは短い日本語または英語のみにする。下部には後から金句を追加できる空白の一行を残す。全体は技術ブログ中の手描きメカニズム図であり、製品 UI や正式なアーキテクチャ図ではない。要素は明瞭で、階層と矢印関係をはっきりさせる。
