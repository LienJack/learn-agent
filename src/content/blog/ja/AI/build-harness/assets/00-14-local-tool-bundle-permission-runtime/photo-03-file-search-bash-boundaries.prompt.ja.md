1. 図解タイプ
Input-process-output。ファイル、検索、端末という三種のローカルツールについて、入力、リスク、制御点を並べて比較する。

2. 画面要素リスト
三つの並行行：ファイルツール、検索ツール、端末ツール。各行に入力、主なリスク、制御点、出力を置く。ファイル行は readFileState、検索行は範囲と予算、端末行は権限と sandbox を強調し、三つの制御点をハイライトする。

3. 正向き画像プロンプト
記事内の技術解説図を描く。テーマは「ファイルツール、検索ツール、端末ツールの境界差」。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線幅はわずかに不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向けの手描きフロー図、明瞭で抑制され、エンジニアリングのラフスケッチ感がある。

構図：三本の横方向スイムレーン式 input-process-output 図。第一行 Files：左にパスカード、中央に readFileState の小さなファイル箱、右に diff/content observation。第二行 Search：左に glob/grep の虫眼鏡、中央に範囲境界と結果予算の漏斗、右に候補リスト。第三行 Terminal：左にコマンドライン窓、中央に権限盾と sandbox 箱、右に exit code と output preview。各行は左から右へ手描き矢印でつなぎ、行間は薄い破線で分ける。readFileState、Result Budget、sandbox の三つの制御点を淡い黄色で強調する。

背景にはごく薄い回路線、ノード接続線、エンジニアリングスケッチの補助線を入れるが、主題より目立たせない。

文字：ノードラベルは短い日本語または英語のみにする。下部には後から金句を追加できる空白の一行を残す。全体は技術ブログ中の手描きメカニズム図であり、製品 UI や正式なアーキテクチャ図ではない。要素は明瞭で、階層と矢印関係をはっきりさせる。
