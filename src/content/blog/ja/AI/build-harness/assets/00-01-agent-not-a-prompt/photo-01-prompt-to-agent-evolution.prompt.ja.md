1. 図解タイプ
横方向の進化パイプライン。この図は Prompt から Harness まで責任が広がることを説明するもので、単一コンポーネントを見せるものではないため。

2. 画面要素リスト
5 つの主要ノード：Prompt、ChatBot、Agent、Harness、実タスク。各ノードは短い日本語ラベルと簡単なアイコンで表す：文書、会話吹き出し、ループ歯車、制御パネル、プロジェクトフォルダ。Agent と Harness を強調する。背景はごく淡いエンジニアリング接続線。下部に一行の余白を名言用に残す。

3. 正向画像提示詞
「Agent は一文の Prompt ではなく、責任が段階的に増えていく実行システムである」というテーマの、記事内向け技術解説図を描く。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線幅は少し不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向けの手描きフローチャート、明快で抑制され、エンジニアリングスケッチ感がある。

構図：横方向パイプライン。左から右へ 5 つの重要ノードを描き、手描き矢印でつなぐ。① Prompt：短いルールが書かれた紙。回答だけを制約することを示す。② ChatBot：2 つの会話吹き出し。複数ターンのメッセージ管理を示す。③ Agent：ループ矢印が小さな歯車を囲む。Loop、Tools、State が組み合わさってタスクを進めることを示し、淡い黄色で強調する。④ Harness：ガードレール付きの制御パネル。権限、ログ、検証、復旧を示し、淡い黄色で強調する。⑤ 実タスク：プロジェクトフォルダとテストマーク。外部環境にある実行可能タスクを示す。

背景にはごく淡い回路線、ノード接続線、エンジニアリングスケッチの補助線を置くが、主題を邪魔しない。各ノードには短い日本語ラベルのみを置く。下部に、後から手書き風の日本語の一文を追加できるスペースを残す。全体は技術ブログ内の手描きメカニズム図のようにし、製品 UI や正式なアーキテクチャ図にはしない。

4. 負向提示詞
写真写実にしない、3D にしない、複雑な UI スクリーンショットにしない、大量のコードを入れない、密集した小さな文字を入れない、複雑な表を入れない、サイバーパンクにしない、ネオン色にしない、暗い背景にしない、カラフルな漫画調にしない、複雑な影を入れない、装飾を増やしすぎない、主要ノードを 5 個より多くしない、背景の回路線を主体より目立たせない、長い日本語文を生成しない。
