1. 図解タイプ
決定経路図。この図は、モデルの提案が直接実行されるのではなく、制御システムによって遮断・検証されることを表すため。

2. 画面要素リスト
左側にモデル提案、中央に Harness 制御ゲート、右側に実行。ゲートには権限、サンドボックス、ログ、検証の 4 つの短いノードを含める。Harness ゲートと検証ノードを強調する。矢印は許可と拒否の 2 経路を持つ。

3. ポジティブ画像プロンプト
「モデルが行動を提案し、Harness がそれを安全に実地へ落とす方法を決める」というテーマの、記事内向け技術解説図を描く。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線幅は少し不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向けの手描きフローチャート、明快で抑制され、エンジニアリングスケッチ感がある。

構図：左から右への決定経路。左側に「モデル提案」：小さなモデルチップが action card を出している。中央に大きめの Harness 制御ゲートを描き、ガードレール付きの検問所のようにして淡い黄色で強調する。ゲート内には 4 つの小ノード：① 権限：鍵アイコン。② サンドボックス：境界フレームアイコン。③ ログ：巻物アイコン。④ 検証：テストチェックアイコン。検証ノードも淡い黄色で強調する。右側に「実行」：端末ウィンドウとプロジェクトフォルダ。さらに細い矢印を下へ伸ばして「拒否/質問」へつなぎ、小さな標識で高リスク動作が止められることを示す。

背景はごく淡い回路線とスケッチ補助線に保つ。日本語ラベルはできるだけ短くする。下部に空白の名言スペースを残す。全体はセキュリティポスターではなく、エンジニアリングシステムの解説図にする。
