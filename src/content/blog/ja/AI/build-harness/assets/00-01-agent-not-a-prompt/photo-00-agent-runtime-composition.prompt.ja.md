1. 図解タイプ
入力-処理-出力図。この図は、Agent が単一の Prompt ではなく、5 つの実行部品が組み合わさった制御プロセスであることを説明するため。

2. 画面要素リスト
左側に 5 つの入力モジュール：モデル、ループ、ツール、状態、制御。中央に淡い黄色で強調した Agent 実行ワークベンチ。右側にタスクフォルダとチェックマークを置き、現実のタスクが進むことを示す。中央の Agent と制御モジュールを強調する。背景にはごく淡いエンジニアリング接続線を使う。

3. ポジティブ画像プロンプト
「Agent はモデル、ループ、ツール、状態、制御システムを組み合わせた実行プロセスである」というテーマの、記事内向け技術解説図を描く。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線幅は少し不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向けの手描きメカニズム図、明快で抑制され、エンジニアリングスケッチ感がある。

構図：input-process-output。画面左側を 5 つの独立した入力モジュールに分け、手描きカードまたはパズルピースのようにする。それらが中央の淡い黄色で強調された「Agent 実行プロセス」ワークベンチに流れ込み、右側の出力は「実タスク進行」を表すプロジェクトフォルダとチェックマークにする。手描き矢印で、5 つの部品が一文の prompt ではなく、共同で実行プロセスを駆動することを示す。

ノード：
① モデル：小さな脳/チップのアイコン。次の一手を判断することを示す。
② ループ：円形矢印アイコン。複数ターンで進めることを示す。
③ ツール：レンチアイコン。外部世界に触れることを示す。
④ 状態：重なった付箋/ログアイコン。プロセスが途切れないことを示す。
⑤ 制御：ガードレール/盾アイコン。権限、予算、検証を示す。
⑥ Agent 実行：中央のワークベンチ。淡い黄色で強調し、組み合わされた制御プロセスを示す。
⑦ タスク：右側のプロジェクトフォルダ + テスト成功チェック。現実のタスクが進むことを示す。

ハイライト：淡い黄色で中央の「Agent 実行」と「制御」モジュールを強調する。

背景：ごく淡い回路線、ノード接続線、エンジニアリングスケッチの補助線。主題を邪魔しない。

文字：非常に短い日本語ノードラベルのみを使う：モデル、ループ、ツール、状態、制御、Agent、タスク。長い日本語文を生成しない。全体は技術ブログ内の手描きメカニズム図のようにし、製品 UI や正式なアーキテクチャ図にはしない。

4. ネガティブプロンプト
