1. 図解タイプ
円形ループ図。核心は、モデルが各ターンで新しい観察に基づいてタスクを進め続けることにあるため。

2. 画面要素リスト
中央に Agent 制御コア。外周に 6 つのノード：現在の現場、モデル判断、行動意図、ツール実行、観察結果、状態更新。ツール実行と状態更新を強調する。矢印で閉ループを作り、右側に外部プロジェクトフォルダを描いて現実世界を表す。

3. 正向画像提示詞
「Agent loop が単発の回答を継続的な前進へ変える仕組み」というテーマの、記事内向け技術解説図を描く。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線幅は少し不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向けの手描きフローチャート、明快で抑制され、エンジニアリングスケッチ感がある。

構図：円形フロー図。画面中央に簡潔な Agent 制御コアのアイコンを置く。小さな制御盤または円形歯車のようにする。外周に時計回りで 6 つのノードを描く：① 現場：フォルダと小さな虫眼鏡。現在のタスクコンテキストを示す。② 判断：脳型の線画またはモデルチップ。モデルが次の一手を決めることを示す。③ 意図：小さな紙片と矢印。tool call または action proposal を示す。④ 実行：レンチと端末ウィンドウ。ツールが実際に実行されることを示し、淡い黄色で強調する。⑤ 観察：ログ巻物。ツール出力を示す。⑥ 更新：重なった状態カード。結果を state に書き戻すことを示し、淡い黄色で強調する。右側に淡いプロジェクトフォルダを加え、ループ外の現実世界にアクセスすることを示す。

背景にはごく淡い回路線とエンジニアリングスケッチの補助線を置く。ノードラベルは短く、長文を避ける。下部に一行の空白の手書き名言スペースを残す。全体は技術ブログ内の手描きメカニズム図のようにする。
