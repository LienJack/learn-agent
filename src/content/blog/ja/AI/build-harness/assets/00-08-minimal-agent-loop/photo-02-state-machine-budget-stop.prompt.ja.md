1. 図解タイプ

state sequence。この図は Agent Loop を、無思考な while true ではなく、状態遷移と停止境界を持つ runtime として説明するため。

2. 画面要素リスト

横方向の状態カード：Ready、Thinking、Acting、Observing、Finished、Stopped。各カードに簡単なアイコンを付ける。予算カウンターと停止標識をハイライト要素にする。Thinking から Acting または Finished への分岐、および Observing から Ready への回帰を矢印で示す。

3. 正向画像提示詞

最小 Agent Loop の状態機械と停止境界をテーマにした、記事内向け技術解説図を描く。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線幅は少し不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向けの手描きフローチャート、明快で抑制され、エンジニアリングスケッチ感がある。

構図：状態シーケンス図。画面に左から右へ 6 枚の状態カードを並べ、手描き矢印でつなぐ：① Ready：状態台帳アイコン。現在の現場が準備済みであることを示す。② Thinking：チップ + 思考吹き出し。モデルが次の一手を判断することを示す。③ Acting：レンチ + ツールカード。制御されたツール実行を示す。④ Observing：ログ紙 + 目。ツール結果を整理することを示す。⑤ Finished：小旗 + チェック。自然終了を示す。⑥ Stopped：停止標識 + カウンター。淡い黄色で強調し、予算、ターン数、中断、エラーによる停止を示す。

ハイライト：淡い黄色で Stopped 状態、予算カウンター、および Observing から Ready へ戻る矢印を強調する。

背景：ごく淡い回路線、ノード接続線、エンジニアリングスケッチの補助線。主題を邪魔しない。

文字：ノードラベルは短くする。下部に手書き名言スペースとして「続けられることと、止まれること」と入れられる一行を残す。日本語文字が不安定なら、後から追加できるよう空白領域だけ残す。

全体は技術ブログ内の手描きメカニズム図のようにし、製品 UI や正式なアーキテクチャ図にはしない。要素は明快に、階層は明確に、矢印関係も明確にする。

4. 負向提示詞

写真写実にしない、3D にしない、複雑な UI スクリーンショットにしない、大量のコードを入れない、密集した小さな文字を入れない、複雑な表を入れない、サイバーパンクにしない、ネオン色にしない、暗い背景にしない、カラフルな漫画調にしない、複雑な影を入れない、装飾を増やしすぎない、主要ノードを 8 個より多くしない、背景の回路線を主体より目立たせない、長い日本語文を生成しない。
