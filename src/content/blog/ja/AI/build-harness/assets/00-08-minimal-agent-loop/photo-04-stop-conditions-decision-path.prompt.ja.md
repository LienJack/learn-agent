1. 図解タイプ

decision path。この図は、Agent Loop が各ターンを続ける前に停止条件チェックを通る必要があることを説明するため。

2. 画面要素リスト

左側入口は Loop Tick。中央の分岐は aborted、maxTurns、budget、model final、invalid intent、execute tool。右側はそれぞれ Finished、Stopped、Continue へ向かう。maxTurns と budget の 2 つの保護ゲートを強調する。

3. 正向画像提示詞

最小 Agent Loop の停止条件がシステムを無限ループから守る仕組みをテーマにした、記事内向け技術解説図を描く。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線幅は少し不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向けの手描きフローチャート、明快で抑制され、エンジニアリングスケッチ感がある。

構図：決定経路図。左側は Loop Tick 入口で、ループ矢印と小さな時計で新ターン開始を表す。中央に 6 つのひし形または小さなゲートノードを描き、手描き矢印でつなぐ：① Abort?：停止ボタン。ユーザー中断を示す。② Max Turns?：カウンターゲート。淡い黄色で強調し、ターン上限を示す。③ Budget?：電量計または財布。淡い黄色で強調し、token、時間、費用の予算を示す。④ Model Final?：小旗。自然完了を示す。⑤ Valid Intent?：構造化カード + 虫眼鏡。ツール意図の検証を示す。⑥ Execute Tool：レンチ + 端末。行動継続を示す。右側に 3 つの出口：Finished、Stopped、Continue を置く。

ハイライト：淡い黄色で Max Turns、Budget、および Stopped へ向かう保護矢印を強調する。

背景：ごく淡い回路線、ノード接続線、エンジニアリングスケッチの補助線。主題を邪魔しない。

文字：ノードラベルは短くする。下部に手書き名言スペースとして「Loop の専門性は、どう止まるかに隠れている」と入れられる一行を残す。日本語文字が不安定なら、後から追加できるよう空白領域だけ残す。

全体は技術ブログ内の手描きメカニズム図のようにし、製品 UI や正式なアーキテクチャ図にはしない。要素は明快に、階層は明確に、矢印関係も明確にする。

4. 負向提示詞

写真写実にしない、3D にしない、複雑な UI スクリーンショットにしない、大量のコードを入れない、密集した小さな文字を入れない、複雑な表を入れない、サイバーパンクにしない、ネオン色にしない、暗い背景にしない、カラフルな漫画調にしない、複雑な影を入れない、装飾を増やしすぎない、主要ノードを 8 個より多くしない、背景の回路線を主体より目立たせない、長い日本語文を生成しない。
