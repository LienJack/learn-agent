1. 図解タイプ

circular loop。この図は、最小 Agent Loop が単発の回答を、現実の観察を中心に継続的に進む閉ループへ変える仕組みを説明するため。

2. 画面要素リスト

中央に「小型 CLI Agent」制御コア。円形ノードはユーザー目標、Think、Act、Tool、Observe、Final。淡い黄色で Observe から Think へ戻る矢印と Act ノードを強調する。右側に小型プロジェクトフォルダとテスト失敗マークを置く。下部に一文の名言スペースを残す。

3. 正向画像提示詞

最小 Agent Loop が単発の回答を複数ステップの行動へ変える仕組みをテーマにした、記事内向け技術解説図を描く。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線幅は少し不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向けの手描きフローチャート、明快で抑制され、エンジニアリングスケッチ感がある。

構図：円形フロー図。画面中央に簡潔な CLI Agent 制御コアを置き、小さな端末ウィンドウとループ矢印の組み合わせのようにする。中心の周囲に 6 つの重要ノードを描き、時計回りの手描き矢印でつなぐ：① ユーザー目標：会話吹き出し + 小旗。「失敗テストを修正する」ことを示す。② Think：チップ + 虫眼鏡。どの事実が不足しているかを判断することを示す。③ Act：レンチ + 構造化カード。淡い黄色で強調し、モデルがツール意図を提示することを示す。④ Tool：フォルダ + 端末。ファイル読み取りやテスト実行を示す。⑤ Observe：ログ紙片 + 目。淡い黄色で強調し、結果の回填を示す。⑥ Final：チェックマーク + 短いレポート。最終結論を示す。

ハイライト：淡い黄色で Act、Observe、および Observe から Think へ戻る矢印を強調する。

背景：ごく淡い回路線、ノード接続線、エンジニアリングスケッチの補助線。主題を邪魔しない。

文字：ノードラベルは短くする。下部に手書き名言スペースとして「観察結果が次の判断を変える」と入れられる一行を残す。日本語文字が不安定なら、後から追加できるよう空白領域だけ残す。

全体は技術ブログ内の手描きメカニズム図のようにし、製品 UI や正式なアーキテクチャ図にはしない。要素は明快に、階層は明確に、矢印関係も明確にする。

4. 負向提示詞

写真写実にしない、3D にしない、複雑な UI スクリーンショットにしない、大量のコードを入れない、密集した小さな文字を入れない、複雑な表を入れない、サイバーパンクにしない、ネオン色にしない、暗い背景にしない、カラフルな漫画調にしない、複雑な影を入れない、装飾を増やしすぎない、主要ノードを 8 個より多くしない、背景の回路線を主体より目立たせない、長い日本語文を生成しない。
