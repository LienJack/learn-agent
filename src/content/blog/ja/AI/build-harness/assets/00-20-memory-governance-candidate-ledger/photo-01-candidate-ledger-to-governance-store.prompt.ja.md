1. 図解タイプ
横向きパイプライン図。要点は、候補記憶が session evidence から candidate ledger に入り、ガバナンスチェックを経て governance store に入る順序を示すこと。

2. 画面要素リスト
画面は左から右へ 6 個のノード：Session イベント、候補抽出、Candidate Ledger、ガバナンスチェック、Review Gate、Governance Store。Session イベントはイベントログアイコン、候補抽出はファネルアイコン、Candidate Ledger は台帳アイコン、ガバナンスチェックは盾と虫眼鏡、Review Gate はゲートアイコン、Governance Store はラベル付きデータベース。淡い黄色で Candidate Ledger と Governance Store を強調。下部にフレーズ用スペース。

3. ポジティブ画像プロンプト
記事内の技術解説図を描く。テーマは Agent の長期記憶が candidate ledger から governance store へ書き込まれるガバナンスチェーン。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線の太さは少し不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向け手描きフロー図、明快で抑制され、エンジニアリングスケッチ感がある。

構図：横向きパイプライン図。左から右へ 6 個の重要ノードを描き、手描き矢印で接続する。① Session イベント：イベントログ紙帯アイコン、ユーザーメッセージ、ツール観測、検証結果を表す。② 候補抽出：ファネルと小ラベル、再利用される可能性のある記憶候補だけを抽出することを表す。③ Candidate Ledger：開いた台帳アイコン、淡い黄色で強調し、pending candidates を表す。④ ガバナンスチェック：盾 + 虫眼鏡、出典、信頼度、範囲、TTL、衝突チェックを表す。⑤ Review Gate：小さなゲートと承認チェック、人手またはポリシーによるレビューを表す。⑥ Governance Store：バージョンラベル付きデータベース、淡い黄色で強調し、監査可能な長期記憶を表す。

ハイライト：Candidate Ledger、Governance Store、ガバナンスチェックから Review Gate への矢印を淡い黄色で強調。

背景：ごく薄い回路線、ノード接続線、エンジニアリングスケッチ補助線を入れ、主題より目立たせない。

文字：ノードラベルは短くする。下部に手書きの日本語フレーズ用スペースを残してよい：「候補は事実ではない。承認後に記憶になる」。日本語文字が不安定なら、後入れ用の空白領域だけを残す。

全体は技術ブログ内の手描きメカニズム図の雰囲気で、製品 UI でも正式なアーキテクチャ図でもない。要素は明快で、階層と矢印関係がはっきりしている。

4. ネガティブプロンプト
写真写実にしない、3D にしない、複雑な UI スクリーンショットにしない、大量のコードを入れない、密集した小文字を入れない、複雑な表にしない、サイバーパンクにしない、ネオン色にしない、暗い背景にしない、カラフルな漫画調にしない、複雑な影を入れない、装飾を増やしすぎない、主要ノードは 8 個を超えない、背景の回路線を主体より目立たせない、長い日本語文章を生成しない。
