1. 図解タイプ
決定経路図。要点は、候補記憶が出典、範囲、TTL、衝突、review gate の分岐判断を通らなければならないこと。

2. 画面要素リスト
画面中央に上から下への判断経路：候補記憶、出典チェック、範囲チェック、TTL チェック、衝突チェック、Review Gate、書き込みまたは拒否。横に 3 本の分岐：信頼度を下げる、追加証拠を待つ、拒否して痕跡を残す。盾、時計、地図ピン、衝突の稲妻、ゲート、データベースなどのアイコン。淡い黄色で Review Gate と衝突チェックを強調。

3. ポジティブ画像プロンプト
記事内の技術解説図を描く。テーマは Memory Governance が候補記憶に対して行うガバナンスチェック経路。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線の太さは少し不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向け手描き決定図、明快で抑制され、エンジニアリングスケッチ感がある。

構図：決定経路図。画面上部に「候補記憶」カードを置き、下へ連続する判断ノードへ手描き矢印で接続する。ノードは：① 出典：小さな印章と証拠紙片、出典の信頼性を表す。② 範囲：地図ピンとフォルダ、user / repo / task scope を表す。③ TTL：小さな時計、有効期限と再レビュー時刻を表す。④ 衝突：ぶつかる 2 枚のカードと小さな稲妻、淡い黄色で強調し、既存記憶との衝突を表す。⑤ Review Gate：ゲートと人間承認の小アイコン、淡い黄色で強調。⑥ Store / Reject：片側はデータベース、片側はアーカイブ箱、承認書き込みまたは拒否して痕跡を残すことを表す。

ハイライト：衝突チェックと Review Gate を淡い黄色で強調。

背景：ごく薄い回路線、ノード接続線、エンジニアリングスケッチ補助線を入れ、主題より目立たせない。

文字：ノードラベルは短くする。下部に手書きの日本語フレーズ用スペースを残してよい：「似ていることと信頼できることは違い、信頼できることと使えることも違う」。日本語文字が不安定なら、後入れ用の空白領域だけを残す。

全体は技術ブログ内の手描きメカニズム図の雰囲気で、製品 UI でも正式なアーキテクチャ図でもない。要素は明快で、階層と分岐関係がはっきりしている。

4. ネガティブプロンプト
写真写実にしない、3D にしない、複雑な UI スクリーンショットにしない、大量のコードを入れない、密集した小文字を入れない、複雑な表にしない、サイバーパンクにしない、ネオン色にしない、暗い背景にしない、カラフルな漫画調にしない、複雑な影を入れない、装飾を増やしすぎない、主要ノードは 8 個を超えない、背景の回路線を主体より目立たせない、長い日本語文章を生成しない。
