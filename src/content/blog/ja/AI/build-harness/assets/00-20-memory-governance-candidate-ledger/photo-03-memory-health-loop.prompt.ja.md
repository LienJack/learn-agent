1. 図解タイプ
環状ループ図。要点は、governance store が終点ではなく、利用シグナル、健康診断、期限切れ、撤回、統合、レビューを継続するガバナンスループを必要とすること。

2. 画面要素リスト
画面中央は Governance Store データベース。外周に時計回りで 6 個のノード：検索利用、利用シグナル、健康チェック、期限切れ/降格、衝突レビュー、バージョン統合。データベース、虫眼鏡、心電図、時計、ゲート、統合矢印などのアイコン。淡い黄色で健康チェックと衝突レビューを強調。下部にフレーズ用スペース。

3. 正向き画像プロンプト
記事内の技術解説図を描く。テーマは Agent 長期記憶のガバナンスストアにおける健康チェックループ。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線の太さは少し不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向け手描きループ図、明快で抑制され、エンジニアリングスケッチ感がある。

構図：環状ループ図。中央に Governance Store とラベル付けされたデータベースと小さな台帳の組み合わせアイコンを置く。外周に時計回りで 6 個のノードを置き、手描き矢印で閉ループにする：① 検索利用：虫眼鏡と query 吹き出し、memory が呼び出されることを表す。② 利用シグナル：小さな統計点と目のアイコン、ヒット、無視、助けた、誤導したなどを表す。③ 健康チェック：心電図とチェックリスト、淡い黄色で強調。④ 期限切れ/降格：時計と下向き矢印、expired / deprecated を表す。⑤ 衝突レビュー：衝突する 2 枚のカードとゲート、淡い黄色で強調。⑥ バージョン統合：ブランチ統合矢印とバージョンラベル、merge / supersede を表す。

ハイライト：健康チェックと衝突レビューを淡い黄色で強調。

背景：ごく薄い回路線、ノード接続線、エンジニアリングスケッチ補助線を入れ、主題より目立たせない。

文字：ノードラベルは短くする。下部に手書きの日本語フレーズ用スペースを残してよい：「長期記憶は追加するだけでなく、保守するもの」。日本語文字が不安定なら、後入れ用の空白領域だけを残す。

全体は技術ブログ内の手描きメカニズム図の雰囲気で、製品 UI でも正式なアーキテクチャ図でもない。要素は明快で、階層と循環関係がはっきりしている。

4. ネガティブプロンプト
写真写実にしない、3D にしない、複雑な UI スクリーンショットにしない、大量のコードを入れない、密集した小文字を入れない、複雑な表にしない、サイバーパンクにしない、ネオン色にしない、暗い背景にしない、カラフルな漫画調にしない、複雑な影を入れない、装飾を増やしすぎない、主要ノードは 8 個を超えない、背景の回路線を主体より目立たせない、長い日本語文章を生成しない。
