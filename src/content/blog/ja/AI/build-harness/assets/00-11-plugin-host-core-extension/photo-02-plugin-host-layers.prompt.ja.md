1. 図解タイプ
レイヤードアーキテクチャ図。外部プラグイン、Plugin Host、Core Kernel の三層間の責任境界を示す。

2. 画面要素リスト
3 つの横方向レイヤー：外部プラグイン層、Plugin Host 層、Core Kernel 層。主ノードは Manifest、Loader、Lifecycle、Registry、Hook Kernel、Contracts、Runtime、Event Log。Registry と Hook Kernel を強調する。アイコンは説明書、漏斗、時計、台帳、ゲート、契約紙、ループ矢印、ログ巻物。

3. 正向き画像プロンプト
記事内の技術解説図を描く。テーマは「Plugin Host の五つの核心部品が外部プラグインと Core Kernel を隔てる」。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線幅はわずかに不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向けの手描きフロー図、明瞭で抑制され、エンジニアリングのラフスケッチ感がある。

構図：三層の横方向レイヤードアーキテクチャ。最上層のタイトルは「外部プラグイン」で、Manifest 説明書、Implementation 小歯車の 2 つのパズル片を置く。中間層のタイトルは「Plugin Host」で、Loader 漏斗、Lifecycle 時計、Registry 台帳、Hook Kernel ゲート、Error Isolation 隔離箱の 5 モジュールを矢印でつなぐ。Registry と Hook Kernel は淡い黄色で強調する。最下層のタイトルは「Core Kernel」で、Contracts 契約紙、Runtime ループ矢印、Event Log ログ巻物の 3 つの安定モジュールを置く。

矢印は上層から中層へ入り、中層から下層へ進むものだけにする。外部プラグインから core へ直接つながる線は描かない。背景には薄い回路線、ノード接続線、工程スケッチ補助線。各ノードには短いラベルのみ。下部に空白の金句スペースを残す。全体は手描きの工程レイヤー図として、責任境界と制御された入口を強調する。
