1. 図解タイプ
レイヤードアーキテクチャ図。焦点は Model 層と Runtime 層の責任境界にある。

2. 画面要素リスト
上層は Model で、申請書だけを書ける。中層は Harness Runtime で、validate、permission、scheduler を含む。下層は現実世界で、ファイルシステム、Shell、Git、テストを含む。境界線は太い黒線で表し、Runtime と境界線を淡い黄色で強調する。

3. ポジティブ画像プロンプト
記事内の技術解説図を描く。テーマは「モデルは申請書を提出し、Runtime だけが外部世界に触れられる」。

画風：アイボリーの紙背景、黒い手描きサインペンの線画、線幅はわずかに不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向けの手描きフロー図、明瞭で抑制され、エンジニアリングのラフスケッチ感がある。

構図：三層の layered architecture。最上層は Model。脳型チップと Intent と書かれた小さな申請書を描く。中間層は Harness Runtime。コンソールと Validate、Permission、Scheduler の 3 つの小モジュールを置き、層全体を淡い黄色で軽く強調する。中間層の下に太い黒い境界線を描き、「実行境界」と短くラベル付けする。最下層は現実世界で、フォルダ、端末ウィンドウ、Git ブランチ、テストのチェック/バツの 4 つの小アイコンを置き、ファイルシステム、Shell、Git、テスト環境を示す。

Model から Runtime へ渡る矢印は申請書のみ。Runtime から現実世界へだけ実行矢印を描く。背景には薄い回路線と工程スケッチ補助線。ラベルは短く、長文にしない。全体で「モデルが直接ツールを所有するのではなく、システムが実行権を持つ」ことを表す。
