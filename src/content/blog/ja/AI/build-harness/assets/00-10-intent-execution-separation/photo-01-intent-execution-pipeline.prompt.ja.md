画風：アイボリーの紙背景、黒い手描きサインペンの線画、線幅はわずかに不均一、少量の淡い黄色ハイライト、editorial technical illustration、技術ブログ向けの手描きフロー図、明瞭で抑制され、エンジニアリングのラフスケッチ感がある。

構図：横方向パイプライン。左から右へ 7 個のノードを並べ、手描き矢印でつなぐ。① ユーザー目標：対話バブルと小旗で「テスト失敗を修正」を表す。② Model：脳型チップで、次の一手の提案だけを担う。③ Intent：申請書アイコン、淡い黄色で強調し、構造化された意図を表す。④ Validate：虫眼鏡付きチェックリストで schema と runtime 検証を表す。⑤ Approve：ゲートまたは盾、淡い黄色で強調し、allow / ask / deny を表す。⑥ Execute：レンチと端末ウィンドウでシステム実行を表す。⑦ Observe：ログ紙とループ矢印で、結果が次のモデル回へ戻ることを表す。

右側の Observe から Model へ弧状矢印を描き、Agent Loop を示す。背景にはごく薄い回路線と工程スケッチ補助線。各ノードには短いラベルのみを置き、下部に空白の金句スペースを残す。全体は技術ブログ内の手描きメカニズム図であり、正式な企業アーキテクチャ図ではない。
